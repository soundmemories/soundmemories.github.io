OAuth2实现密码哈希与Bearer JWT令牌验证

1.JWT 即 JSON Web Tokens。JWT 是一种将 JSON 对象编码为没有空格，且难以理解的长字符串的标准。

JWT 字符串没有加密，任何人都能用它恢复原始信息，但 JWT 使用了签名机制，接受令牌时，可以用签名校验令牌。
JWT 令牌具有时限性，过期后令牌失效，如果用户篡改令牌的过期时间，因为签名不匹配会导致身份验证失败。
JWT 需安装pip install pyjwt。若使用类似 RSA 或 ECDSA 的数字签名算法，应该安装加密库依赖项pip install pyjwt[crypto]。

2.哈希是指把特定内容（本例中为密码）转换为乱码形式的字节序列（其实就是字符串）。
每次传入完全相同的内容时（比如，完全相同的密码），返回的都是完全相同的乱码。
但这个乱码无法转换回传入的密码。
需安装处理密码哈希的包，本例推荐算法是Bcrypt，所以安装pip install passlib[bcrypt]。

后台任务

有时会遇到需要在请求处理完成后执行一些任务的情况，比如发送邮件、清理缓存等。FastAPI 提供了 BackgroundTasks 类来处理这种情况。
注意，若后台任务非常复杂和繁重，应当选择专用的"异步任务队列"，如Celery，再配合一个"消息代理"（），如RabbitMQ。
Celery：管理分配任务到不同的服务器，并取得结果。不管和服务器之间的通信。
RabbitMQ：消息代理，用于Celery和服务器之间的通信。

1.比如收到的文件必须经过一个缓慢的过程，可以先返回一个"Accepted"(HTTP 202)响应，并在后台处理这个任务。
