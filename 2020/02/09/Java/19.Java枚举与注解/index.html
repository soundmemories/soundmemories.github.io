Java枚举与注解

发表于 2020-02-09 分类于 Java 本文字数： 3.4k 阅读时长 ≈ 12 分钟

枚举

枚举是一种类，是一种特殊的类，特殊在它的对象是有限个常量对象。
某些类的对象只有确定的有限个时，可以把这样的类声明为枚举类。
比如，星期：Monday(星期一)......Sunday(星期天)。

JDK1.5前枚举创建（参考TestEnum）：
（1）私有化类的构造器，保证不能在类的外部创建其对象。
（2）在类的内部创建枚举类的实例。声明为：public static final。

JDK1.5后枚举创建（参考TestEnum2及其之后示例）：
（1）使用 enum 定义的枚举类，默认继承了java.lang.Enum类。因此不能再继承其他类。
（2）枚举类的所有构造器只能使用 private 访问控制符。
（3）枚举类的所有实例必须在枚举类的首行显式列出(逗号分隔，分号结尾)，它们实际上都是public static final修饰的常量对象。
（4）JDK 1.5 之后可以在 switch 表达式中使用Enum定义的枚举类的对象作为表达式, case 子句可以直接使用枚举值的名字。

java.lang.Enum是所有 Java 语言枚举类型的公共基本类。
"protected Enum(String name, int ordinal)"：Enum类中唯一的构造器，这个构造器是编译器自动调用，在所有枚举类型的构造器的首行，并且自动传入name和ordinal的值。name：就是常量对象名称。ordinal：就是常量对象的序号，其中初始常量序数为零。
Enum类中的方法：
（1）String name()：返回此枚举常量的名称。
（2）int ordinal()：返回此枚举常量的序号，初始常量序数为零。
（3）String toString()：返回枚举常量的名称。重写了Object类的toString方法。在eclipse中不能自动生成枚举类型的toString方法，但可手动重写，因为eclipse认为，父类重写的toString方法的实现已经很完美了，就是返回常量对象名，已经能代表一切。
（4）int compareTo(Object obj)：按照常量对象的顺序比较。
Enum类API中没有的方法（是编译器帮我们生成的方法）：
（1）枚举类型[] values()：返回一个数组，数组中是所有枚举对象。
（2）枚举类型 valueOf(String name)：返回指定名称的枚举对象。

注解

注解是从JDK5.0开始引入的新特性。
注解也是一种注释，它是代码级别的注释，是用代码注释代码。
注解是以“@注释名”在代码中存在的，还可以添加一些参数值，例如：“@SuppressWarnings(value=”unchecked”)”。

1、一个完整的注解，由三部分组成：
（1）声明
（2）使用：绝大多数程序员，面对的基本上是（2）使用。
（3）读取

2、系统中预定义的三个最基本的注解：
（1）@Override：检查这个方法是否符合重写的要求。

  • 方法名：相同。
  • 形参列表：相同。
  • 返回值类型：基本数据类型和void：相同。引用数据类型：<=。
  • 修饰符：权限修饰符：>=。其他修饰符：private，static，final不能重写。

（2）@SuppressWarnings(xx)：抑制警告。
（3）@Deprecated：标记某个方法、类、属性等已过时。不建议程序员继续使用，因为可能有问题，也可能是不好用。
面向对象的开发原则：对修改关闭，对扩展开放。

3、文档注释（“/**”）：
（1）@author：标明开发该类模块的作者，多个作者之间使用”,“分割。
（2）@version：标明该类模块的版本。
（3）@see：参考转向，也就是相关主题。
（4）@since：从哪个版本开始增加的。
（5）@param： 对方法中某参数的说明，如果没有参数就不能写。
（6）@return： 对方法返回值的说明，如果方法的返回值类型是void就不能写。
（7）@throws/@exception： 对方法可能抛出的异常进行说明，如果方法没有用throws显式抛出的异常就不能写。

其中 @param，@return和@exception这三个标记都是只用于方法的。
@param的格式要求：“@param 形参名 形参类型 形参说明”。
@return的格式要求：“@return 返回值类型 返回值说明”。
@throws/@exception的格式要求：“@exception 异常类型 异常说明”。
@param和@exception可以并列多个。javadoc.exe就是这些注解的信息处理流程。

（1）-encoding <name>：源文件编码名称。
（2）-charset <charset>：用于跨平台查看生成的文档的字符集。
（3）docencoding <name>：指定输出的字符编码。

4、JUnit框架中的注解
JUnit是由 Erich Gamma 和 Kent Beck 编写的一个回归测试框架（regression testing framework），供Java开发人员编写单元测试之用。多数Java的开发环境都已经集成了JUnit作为单元测试的工具。
Junit测试是程序员测试，即所谓白盒测试，因为程序员知道被测试的软件如何（How）完成功能和完成什么样（What）的功能。

如何使用？
因为JUnit不是JDK自带的，所以在eclipse中需要导入：右键自己的Project，Build Path，Add Libraries，选择JUnit即可。

使用要求：
（1）使用JUnit测试的类必须是public的。
（2）JUnit4要求：这些方法都必须是public，无参，无返回值。

JUnit的常用注解：
（1）@Test：标记在非静态的测试方法上。只有标记@Test的方法才能被作为一个测试方法单独测试。一个类中可以有多个@Test标记的方法。运行时如果只想运行其中一个@Test标记的方法，那么选择这个方法名，然后单独运行，否则整个类的所有标记了@Test的方法都会被执行。
@Test(timeout=1000)：设置超时时间，如果测试时间超过了你定义的timeout，测试失败。
@Test(expected)： 申明出会发生的异常，比如 @Test（expected = Exception.class）
（2）@BeforeClass：标记在静态方法上。因为这个方法只执行一次。在类初始化时执行。
（3）@AfterClass：标记在静态方法上。因为这个方法只执行一次。在所有方法完成后执行。
（4）@Before：标记在非静态方法上。在@Test方法前面执行，而且是在每一个@Test方法前面都执行
（5）@After：标记在非静态方法上。在@Test方法后面执行，而且是在每一个@Test方法后面都执行
（6）@Ignore：标记在本次不参与测试的方法上。这个注解的含义就是“某些方法尚未完成，暂不参与此次测试”。
@BeforeClass、@AfterClass、@Before、@After、@Ignore都是配合@Test它使用的，单独使用没有意义。


5、自定义注解

（1）注解的成员变量在注解定义中以无参数方法的形式来声明。其方法名和返回值定义了该成员的名字和类型，我们称为配置参数。类型只能是：八种基本数据类型、String类型、Class类型、enum类型、Annotation类型、以及以上所有类型的数组。
（2）可以在定义注解的成员变量时用 default 关键字为其指定初始值。
（3）如果注解的配置参数只有一个，并且名字是value，那么在使用赋值时，可以省略"value="。
（4）如果定义的注解含有配置参数，那么使用时必须指定参数值，除非它有默认值。格式是“配置参数名 = 参数值”，当配置参数是数组类型时，格式是“配置参数名 = {参数值}”。
（5）读取这个注解时，要获取配置参数的值，可以当成方法一样来访问“自定义注解对象.配置参数()”。

6、元注解
注解的注解，加在注解上面的注解，就叫做元注解。
Java5.0定义了4个标准的元注解类型，它们被用来提供对其它注解类型作说明（在java.lang.annotation包）。
（1）@Target：用于描述注解的使用范围（即被描述的注解可以用在什么地方）。它由ElementType枚举的常量对象来决定：TYPE,FIELD,CONSTRUCTOR等10个范围。
（2）@Retention：定义了该注解在什么范围内有效。它由RetentionPolicy的3个常量对象来指定：SOURCE（源代码阶段.java）, CLASS（字节码阶段.class，默认值）, RUNTIME（运行时JVM）。
（3）@Documented：这个注解应该被javadoc工具记录。默认情况下javadoc是不包括注解的，但如果声明注解时指定了@Documented,则它会被javadoc之类的工具处理。
（4）@Inherited：允许子类继承父类中的注解。
