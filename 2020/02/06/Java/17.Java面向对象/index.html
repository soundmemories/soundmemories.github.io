思想和编程思想

思想：非常高度概括的思想和行为的准则。
编程思想：指引我们编程（开发/写代码）的原则。
面向对象的程序员从执行者的角度转变为指挥者的角度。

面向过程的编程思想：
1.经典代表：C语言
2.程序的最小单位：函数。一个函数代表一个功能。
3.程序员关注的角度：把大的功能分解为一个一个小的功能。
4.面向过程是把数据与功能分开的。重点关注怎么做。

面向对象的编程思想：
1.经典代表：Java语言
2.程序的最小单位：类/对象。
3.程序员关注的角度：从复杂的业务逻辑/功能中先抽取出“类”，然后再考虑这个类能包含什么属性、方法等。
4.面向对象是把数据和功能合起来的。重点关注“谁”做。

继承

继承（Inheritance）：当多个类中存在相同属性和行为时，将这些内容抽取到单独一个类中，那么多个类无需再定义这些属性和行为，只要继承单独那个类即可，提高了代码的复用性。这里多个类指子类，单独的这个类为父类。
比如Teacher、Student等子类，可通过父类Person继承共有的部分，就不需要在子类别中再次去声明。

1.继承的写法？

2.继承的特点：
（1）子类可以继承父类所有的属性和方法，包括私有的。子类不能对继承的私有成员直接进行访问，但可通过继承的公有方法来访问。可以扩展父类没有的属性和方法。
（2）子类不会继承父类的构造器，但是一定会调用父类的构造器。默认调用父类的无参构造，super()写或不写都会调用父类的无参构造，如果要写，必须在子类构造器首行。如果父类没有无参构造，必须在子类的构造器首行使用super(实参列表)显式调用父类的有参构造，否则编译报错。
（3）Java只支持单继承，不允许多重继承，但是支持多层继承。一个子类只能有一个父类，一个父类可以派生出多个子类。Java支持多层继承，代代相传。

3.如果在类的声明中未使用extends关键字指明其父类，类默认继承根父类java.lang.Object。

多态

多态（Polymorphism）：一类事物的多种表现形态。比如，人可分为，男人、女人、人妖。
多态：编译时类型和运行时类型不一致，就出现多态。

1.变量引用形式：
（1）本态引用：左边的变量与右边的对象是同一种类型。
（2）多态引用：左边的变量是父类类型，右边的对象是子类的对象。

2.多态的的特征（编译类型与运行时类型不一致）：
（1）编译时看左边，看的是父类的引用（从父类中确定所调用的方法）。
（2）运行时看右边，看的是子类的对象（实际运行的是子类重写父类的方法）。

3.多态的特点：
（1）要有继承或实现关系。
（2）要有方法的重写，所以和方法有关，和属性无关。
（3）不能访问子类独有的属性和方法，能访问重写的方法。
（5）成员变量不具备多态性，只看引用变量所属的类。
（6）静态方法不能被重写，也没有多态性。

多态的应用：
（1）多态数组：数组声明为父类，实际存储的是父类或子类实例。
（2）多态参数：形参声明为父类，实参是父类或子类实例。
（3）多态属性+多态参数：属性、形参声明是父类；属性的值、实参是父类或子类实例。
注意：属性没有多态，这里“多态属性”指通过“多态参数”最终赋值给属性的意思。

方法

方法（method）又称为函数（function），是一个独立的可复使用的功能。
例如：Math.random()代表产生 \([0,1)\) 随机值的功能，Math.sqrt(x)代表求 \(\sqrt{x}\) 的功能。

方法声明

声明方法前需要明确：

1.声明方法：
（1）必须先声明后使用。
（2）不调用不执行。

2.返回值类型：
（1）void：代表无返回值。不需要但可以有return返回值语句，此时会提前结束方法。
（2）非void：必须有return返回值语句，可以是任意的Java数据类型，包括8种基本数据类型、所有的引用数据类型。

3.形参列表：
（1）()表示无参，(形参列表)表示有参。
（2）有参写法为 (数据类型1 形参名1, 数据类型2 形参名2, ....)。

4.调用：
（1）在本类中，同一个级别，直接调用。

（2）在其他类中，需要通过"对象名.方法"调用。

实参与形参：
1.实参是赋值给形参，要求实参的个数、类型、顺序与形参一一对应。
2.形参为基本数据类型：实参给形参传的是数据值，形参值的修改不影响实参，因为实参是“copy”了一份数据值给形参。
3.形参为引用数据类型：实参赋值给形参的是对象的地址值，若形参修改了对象的属性值，则实参对象的属性值也会修改。
4.陷阱：当形参是引用数据类型，实参给形参地址值之后，形参修改了地址值（比如指向新对象），那么和实参就无关了。

方法重载

方法重载：在同一个类中，出现了两个或多个的方法名称相同，形参列表不同，称为方法的重载。
方法名代表方法的功能，这几个方法的功能是一样的，但输入参数是不同的。而且为了调用者的方便，功能相同的方法取名一样的话，对于调用者来说就减少了学习和记忆的成本。

1.重载的特点：
（1）在同一个类中多个方法；
（2）这些方法名必须相同；
（3）参数列表必须不同（参数的个数、数据类型不同）。
注意：与返回值类型无关！

方法重写

重写（Override）：当父类中的方法，对于子类来说不适用的时，子类可以对父类中的方法进行重写/覆写。

1.重载（Overload）与重写（Override）有什么区别？
（1）重载是在同一个类中，重写是子类与父类中。
（2）方法名：重载和重写都要求名称完全一致。
（3）参数列表：重载必须不同；重写必须相同。
（4）返回值类型：重载和返回值类型无关；重写对基本数据类型，严格要求类型一致。

声明原则：
1.一个方法只能有一个可变参数。

带有可变参数的方法的调用原则：
1.非可变参数，实参的类型，个数，顺序与形参仍然要一一对应。

代码块

一般来说是用于类或实例成员变量的初始化，分为静态代码块和非静态代码块（构造代码块）。

静态代码块

1.可以为类变量（静态属性）初始化。
2.随着类的初始化而执行，且只执行一次。
3.子类初始化时，它的父类没有初始化会先初始化父类。
4.在静态代码中不能使用this和super关键字。
5.如果有重名问题，需要访问本类或父类的静态成员，那么可以使用本类名.xx、父类名.xx，如果没有重名问题，直接访问即可。

补充知识点：

父子类的实例初始化？
1.先父类的初始化<clinit>()，再子类的初始化<clinit>()。
2.static方法不能被重写，因为静态方法是类方法，是属于类的，和对象无关。

4、父子类的实例初始化？
（1）原先说super()和super(实参列表)是调用父类的构造器，现在就要纠正为调用父类实例初始化方法了。

初始化顺序

1.静态：编译器会把 “静态变量的显式初始化”、“静态代码块” 按顺序合并为一个<clinit>的类初始化方法，它只有一个，且只会执行一次，先父类再子类。
3.非静态：编译器会把 “非静态变量的显示初始化”、“非静态代码块” 按顺序合并到每一个构造器中，构成一个个的<init>实例初始化方法，有几个构造器就有几个<init>方法，new一次执行一次，原构造器的内容会最后执行，先父类再子类。
5.不管是 “静态变量” 还是 “非静态变量”，<clinit>或<init>初始化之前，就有默认值。

内部类

当一个类内部，某个部分需要一个完整的结构进行描述，且它只对外部事物提供服务，那么可以使用内部类结构描述，内部类可以使用外部类的所有的成员，包括私有变量。 JRE核心类库集合的迭代器大量使用内部类，可看源代码学习。

成员内部类

2、成员内部类的特点：
（1）仍然是一个独立的类，在编译之后变成独立的.class文件，但是前面冠以外部类名和$符号，比如 Outer$Inner.class。
（2）可以用修饰符public、protected、default、private，还可以用final、abstract。

局部内部类

最少使用的一种，几乎见不着。

1.局部内部类声明：

2.局部内部类特点：
（1）仍然是一个独立的类，在编译之后变成独立的.class文件，外部类的类名$数字编号内部类名.class。
（2）只能在声明它的方法或代码块中使用，而且是先声明后使用。作用域问题，导致除此之外的任何地方都不能使用该类。
（3）和局部变量地位类似，不能用public、protected、private、static，可用abstract或final。
（4）不能用static修饰，即不能包含静态成员。
（5）可用外部类的成员，包括私有的。但是否可以使用外部类的非静态成员要看所在方法是否是非静态的。
（6）可用外部方法的局部变量，但是必须是final的（Java8自动添加final），这是由局部内部类和局部变量的生命周期不同所致，避免局部内部类对象被返回到外部类的外面使用时，访问不到这个局部变量，所以要把这个局部变量变为final的常量。

抽象方法
