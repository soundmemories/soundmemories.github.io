团队人员调度软件

模拟实现一个基于文本界面的《团队人员调度软件》。
熟悉Java面向对象的高级特性，进一步掌握编程技巧和调试技巧。

主要涉及以下知识点：
（1）类的继承和多态；
（2）对象的关联；
（3）static和final修饰符；
（4）特殊类的使用；
（5）异常处理。

需求说明

该软件实现以下功能：
（1）软件启动时，根据给定的数据创建公司部分成员列表（数组）。
（2）根据菜单提示，基于现有的公司成员，组建一个开发团队以开发一个新的项目。
（3）组建过程包括将成员插入到团队中，或从团队中删除某成员，还可以列出团队中现在成员的列表。
（4）开发团队成员包括架构师、设计师和程序员。

本软件采用单级菜单方式工作。

开发团队人员组成要求：
（1）最多一名架构师；
（2）最多两名设计师；
（3）最多三名程序员。

（1）成员已满，无法添加；

（4）该员正在休假，无法添加；
（5）该员工已是团队成员；
（6）团队中只能有一名架构师；
（7）团队中只能有两名设计师；
（8）团队中只能有三名程序员。

提供的数据和工具

项目中提供了TSUtility.java类，可用来方便地实现键盘访问。
该类提供了以下静态方法：
public static char readMenuSelection()：
　　用途：该方法读取键盘，如果用户键入’1’-’4’中的任意字符，则方法返回。返回值为用户键入字符。
public static void readReturn()：
　　用途：该方法提示并等待，直到用户按回车键后返回。
public static int readInt() ：
　　用途：该方法从键盘读取一个长度不超过2位的整数，并将其作为方法的返回值。
public static char readConfirmSelection()：
　　用途：从键盘读取‘Y’或’N’，并将其作为方法的返回值。

项目结构

先建立4个包：
（1）com.memories.bean：用来存放基本的JavaBean。例如员工类等基础数据封装JavaBean。
（2）com.memories.service：service模块为实体对象（Employee及其子类如程序员等）的管理模块，NameListService和TeamService类分别用各自的数组来管理公司员工和开发团队成员对象。
（2）com.memories.util：用来存放工具类。例如键盘输入工具类TSUtility.java。
（4）com.memories.view：用来存放视图类，负责菜单的显示和处理用户操作。例如TeamView.java。
可以再创建test和exception包，用于测试和放异常类。

开发详解

实体类

架构师Architect继承设计师Designer，设计师继承程序员Programmer，程序员继承员工类Employee。
程序员及其子类关联一个设备对象Equipment。即在程序员中有一个设备的引用，或者说有一个Equipment类型的属性。
而设备Equipment又分为三种，都实现了设备接口分别为PC电脑类，NoteBook笔记本类，Printer打印机类。
Employee类及其子类的设计

classDiagram class Employee{ -id:int -name:String -age:int -salary:double +Employee(id: int , name: String, age: int, salary: double) } class Programmer{ -memberId:int -status:Status -equipment:Equipment +Programmer(id: int , name: String, age: int, salary: double, equipment: Equipment) }

memberId：用来记录成员加入开发团队后在团队中的ID。
status是项目自定义的枚举类型，表示成员的状态：FREE-空闲、USY-已加入开发团队、VOCATION-正在休假。
equipment：表示该成员领用的设备。
可根据需要自行为类提供各属性的get/set方法以及重载构造器。
classDiagram class Designer{ -bonus:double +Designer(id: int , name: String, age: int, salary: double,equipment: Equipment,bonus : double) } class Architect{ -stock:int +Architect (id: int , name: String, age: int, salary: double,equipment: Equipment,bonus : double, stock : int) }

bonus：奖金。
stock：公司奖励的股票数量。
可根据需要自行为类提供各属性的get/set方法以及重载构造器。
Equipment接口及其实现子类的设计
classDiagram class Equipment{ <<interface>> +getDescription():String } class NoteBook{ -model:String -price:double +NoteBook(model: String, price: double) } class PC{ -model:String -display:String +PC(model: String, display: String) } class Printer{ -type:String -name:String +Printer(type: String, name: String) }

model：机器的型号。
display：显示器名称。
可根据需要自行为类提供各属性的get/set方法以及重载构造器。

业务类NameListService

classDiagram class NameListService{ -employees:Employee[] +NameListService() +getAllEmployees() Employee[] +getEmployee(int id) Employee }

employees：用来保存所有公司员工对象。
NameListService()构造器：
（1）根据项目提供的Data类构建相应大小的employees数组。
（2）再根据Data类中的数据构建不同的对象，包括Employee、Programmer、Designer和Architect对象，以及相关联的Eqipment子类的对象。
（1）将对象存于数组中。
（1）Data类位于com.memories.team.service包中。
getAllEmployees()方法：获取当前所有员工。返回：包含所有员工对象的数组。
getEmployee(int id) throws TeamException方法：获取指定ID的员工对象。返回：指定员工对象。异常：找不到指定的员工。
另外，可根据需要自行添加其他方法或重载构造器。

业务类TeamService

classDiagram class TeamService{ -counter~int~ +MAX_MEMBER~final~~int~ +team~Programmer[]~ -total~int~ +getTeam() Programmer[] +addMember(e: Employee) void +removeMember(memberId: int) void }

counter：为静态变量，用来为开发团队新增成员自动生成团队中的唯一ID，即memberId。（提示：应使用增1的方式）
MAX_MEMBER：开发团队最大成员数。
team：数组用来保存当前团队中的各成员对象。
total：记录团队成员的实际人数。
getTeam()方法：返回当前团队的所有对象。返回：包含所有成员对象的数组，数组大小与成员人数一致。
addMember(e: Employee) throws TeamException方法：向团队中添加成员。异常：添加失败， TeamException中包含了失败原因。
removeMember(memberId: int) throws TeamException方法：从团队中删除成员。异常：删除失败，TeamException中包含了失败原因。
另外，可根据需要自行添加其他方法或重载构造器。

主界面TeamView

classDiagram class TeamView{ -listSvc~NameListService~ -teamSvc~TeamService~ +team~Programmer[]~ +enterMainMenu() oid -listAllEmployees() void -addMember() void -deleteMember() void +main(args: String[]) void }

listSvc和teamSvc属性：供类中的方法使用。
enterMainMenu ()方法：主界面显示及控制方法。
以下方法仅供enterMainMenu()方法调用：
（1）listAllEmployees ()方法：以表格形式列出公司所有成员。
（2）addMember ()方法：实现添加成员操作。
（3）deleteMember ()方法：实现删除成员操作。

实现步骤

第1步：创建项目基本组件
（1）创建TeamSchedule项目。
（2）按照设计要求，创建所有包。
（3）按照设计要求，创建所有类和接口的声明。
（4）将项目提供的几个类复制到相应的包中。
（5）编写Employee类及其各子类代码。
（6）编写Equipment接口及其各实现子类代码。
（7）检验代码的正确性。

第2步：实现service包中的类
（1）按照设计要求编写sNameListService类
（2）在NameListService类中临时添加一个main方法中，作为单元测试方法。
（3）在方法中创建NameListService对象，然后分别用模拟数据调用该对象的各个方法，以测试是否正确。注：测试应细化到包含了所有非正常的情况，以确保方法完全正确。
（4）重复1-3步，完成TeamService类的开发

第3步：实现view包中类
（1）按照设计要求编写TeamView类，逐一实现各个方法，并编译。
（2）执行main方法中，测试软件全部功能。
