思想和编程思想

思想：非常高度概括的思想和行为的准则。
编程思想：指引我们编程（开发/写代码）的原则。

  • 面向过程的编程思想：
    • 经典代表：C语言
    • 程序的最小单位：函数。一个函数代表一个功能。
    • 程序员关注的角度：把大的功能分解为一个一个小的功能。
    • 面向过程是把数据与功能分开的。重点关注怎么做。
  • 面向对象的编程思想：
    • 经典代表：Java语言
    • 程序的最小单位：类/对象。
    • 程序员关注的角度：从复杂的业务逻辑/功能中先抽取出“类”，然后再考虑这个类能包含什么属性、方法等。
    • 面向对象是把数据和功能合起来的。重点关注“谁”做。

面向对象的程序员从执行者的角度转变为指挥者的角度。

把大象装进冰箱的问题？
面向过程：
（1）把冰箱门打开
（2）把大象装进去
（3）把冰箱门关上
面向对象：
（1）对象：人、冰箱、大象
（2）人把大象装进冰箱

面向对象的学习围绕几条主线：
1、类与对象

类与对象

类：一类具有相同特性的事物的抽象描述。

类与对象的关系：
（1）类是众多对象的抽象的描述，类是对象的设计图，创建对象的模板。
（2）对象是类的个体，具体的实例。

类的设计

类的提取：名词，按现实世界中事物的类别 转成 计算机程序中的类。
类的设计：
　类的成员设计（5大成员/5大组成部分）
　　（1）属性：数据特征
　　（1）方法：行为特征、功能特征
　　（1）构造器：创建对象时用
　　（1）代码块：在类初始化、对象初始化时用
　　（1）内部类：某个大的事物中，包含了一个独立的小事物类别，且一般这个小事物只为大的事物服务时。

方法

方法(method)又称为函数(function)。
方法是代表一个独立的可复用的功能。
例如：

声明方法的好处：
（1）简化代码量：代码可以复用，程序员不用重复造轮子；
（2）方便用户：使用者不管方法的内部如何实现，只要关注这个方法的功能、是否需要传参数、是否有返回值等问题。

1、如何声明一个方法？

方法 = 方法签名/方法头 + 方法体
方法签名/方法头：【修饰符】 返回值类型 方法名(【形参列表】)
方法体：{}

2、返回值类型：
（1）void：代表无返回值。不需要return返回值语句，但可以有return语句，此时会提前结束方法。
（2）非void：可以是任意的Java数据类型，包括8种基本数据类型、所有的引用数据类型。需要return返回值语句。

3、方法名：
（1）见名知意：能够体现这个方法的功能；
（2）和变量名一样，从第二个单词开始首字母大写。

4、(【形参列表】)：
（1）()表示无参，(形参列表)表示有参。
（2）(数据类型 形参名)或(数据类型1 形参名1, 数据类型2 形参名2, ....)。

5、方法的原则：
（1）必须先声明后使用。
（2）不调用不执行。

6、调用：
（1）在本类中，同一个级别，直接调用。

（2）在其他类中，需要通过"对象名.方法"。

实参与形参：实参是赋值给形参，要求实参的个数、类型、顺序与形参一一对应。

形参是基本数据类型：实参给形参传的是数据值，形参值的修改不影响实参，因为实参是“copy”了一份数据值给形参。
形参是引用数据类型：实参赋值给形参的是对象的地址值，若形参修改了对象的属性值，则实参对象的属性值也会修改。

陷阱：当形参是引用数据类型，实参给形参地址值之后，形参修改了地址值（比如指向新对象），那么和实参就无关了。

方法的重载

（1）在同一个类中，出现了两个或多个的方法名称相同，形参列表不同，称为方法的重载。
（2）形参列表不同：形参的个数、数据类型不同。
（3）方法的重载和返回值类型无关。

为什么要有方法的重载？或者换句话说，为什么要声明多个方法名称相同呢？
（1）方法名代表方法的功能，这几个方法的功能是一样的，例如：都是找最大值。
（2）为了调用者的方便，功能相同的方法取名一样的话，对于调用者来说就减少了学习和记忆的成本。

可变参数

JDK1.5之后引入的。
形参列表中出现了“数据类型… 形参名”形式的形参，就是可变参数。

可变参数的声明的原则：
（1）一个方法只能包含一个可变参数。
（2）可变参数必须是形参列表的最后一个。

包含可变参数的方法的调用原则：
（1）可变参数的形参部分，可以传入0~n个对应类型的实参，也可以传入对应类型的数组对象。
（2）非可变参数的形参部分，原来该怎么传，现在还怎么传。

变量的分类

学习完属性和方法，现在可以回头思考下变量的分类，主要为成员变量和局部变量，今天主要讨论非静态变量。

（1）声明的位置不同

（2）运行时在内存中的位置也不相同
成员变量：堆。
局部变量：栈。

（3）初始化不同
成员变量：如果没有初始化，有默认值。
局部变量：如果没有初始化，就会报错，不能用。

（4）生命周期
成员变量：随着对象的创建而分配，随着对象被垃圾回收器回收而结束。
局部变量：方法被调用，开始分配，方法执行结束就结束。

（5）可用的修饰符
成员变量：有很多修饰符，具体哪些后面再讲。
局部变量：唯一的一个就是final，具体什么意思之后再讲。

面向对象的基本特征一：封装

封装是面向对象编程的核心思想。将对象的属性和行为封装起来，其载体就是类。类通常对客户隐藏其实现细节，这就是封装的思想。

1、封装的意义：
（1）隐藏实现细节。
（2）安全。

2、封装的类型：
（1）属性的封装、方法的封装等。
（2）组件的封装。
（3）系统的封装。

3、封装的实现：靠权限修饰符来控制可见的范围。
（1）public：公共的，可用于修饰类、属性、方法。表示在其他任何想要使用的地方都可见。
（2）protected：受保护的，可用于修饰属性、方法。表示在本包或其他包的子类中可见。
（3）缺省的：可用于修饰类、属性、方法。表示在本包下可见。
（4）private：私有的，可用于修饰属性、方法。只能在本类中可见，本类以外的地方不可见。

Java权限修饰符public、protected、private置于类的成员定义前，用来限定对象对该类成员的访问权限。
对于外部类class的权限修饰只可以用public、和default（缺省）。
public类可以在任意地方被访问。
default类只可以被同一个包内部的类访问。

哪些可以加权限修饰符？
类、属性、方法、构造器、内部类都可以加权限修饰符。

分别可以加什么权限修饰符？
类：缺省或public（如果类前面有public，那么必须与源文件名相同）。
属性：四种都可以。
方法：四种都可以。

4、属性的封装
大多数情况下，属性都是private。
如果属性私有化了，我们会提供get/set方法。
