Java程序的开发步骤

  • 编写代码
    • 保存为.java的源代码文件。
  • 编译代码
    • 把.java源文件编译为.class的字节码文件（因为JVM只认识字节码）。
  • 编译工具
    • javac.exe。
    • 把.java编译为.class字节码。
  • 如何使用编译工具：
    • 格式：javac 源文件名.java
  • 运行.class工具
    • java.exe。
  • 如何使用运行工具：
    • 格式：java 字节码文件名 或 类名，不要有后缀。

可能出现的问题

乱码问题

计算机只识别0和1。
人类世界中字符是很丰富的。

最早人类使用计算机时，用硬件机器语言编写程序，即0和1组成的字节码。
为了使用计算机的人方便，开始引入了助记符等，可以表示更丰富的字符等信息。
如ASCII码：数字十进制的65表示字符A，计算机底层把65十进制再转为二进制。最早ASCII码只能表示128字符。
后来计算机不仅仅在美国了，流传到其他国家，各个国家在ASCII码基础上扩展了128个，变成256个了，但是后面扩展的128个不具有通用性。
计算机传入亚洲，就开始出现多字节编码，例如：中国大陆GB2312，后来的GBK，中国台湾等地区Big5等。但越来越多的字符编码出现，使各国之间的文档交流出现问题。

计算机希望在全世界范围能够无障碍的交流，开始引入万国码，Unicode编码表，这个编码表，可以用它的规则表示全世界所有的字符，每一个字符都有自己唯一的编码。
Unicode编码表只说了，字符用什么数字表示，但是这个数字范围跨度很大，有1个字节的，2个字节的，3个字节的，4个字节等，没有统一标准，这在网络传输中出现问题，例如，我收到4个字节的数据时，该表示为4个字符呢还是1个字符呢？
为了解决传输的问题，出现了UTF-8等编码方式，规定了，到底几个字节是一个字符。当然不同的编码方式对同一组数据解析出来的字符是不同的。

修改当前源文件的字符编码：
1. Notepad++ 中选择 【编码】 -> 转为ANSI（与当前操作系统一致）编码。
2. 若想之后建立的源文件，都是与ANSI编码一致的，Notepad++ 中选择 【设置】 -> 【首选项】 -> 【新建】 -> ANSI编码。

大小写问题

  1. 源文件名.java的大小写问题：
    • 源文件名不区分大小写。
  2. 字节码文件名（类名）的大小写问题：
    • 字节码文件名严格区分大小写。
  3. 代码中大小写问题：
    • 代码中严格区分大小写。
所以在作为程序员的时候，一定要严格区分大小写。

源文件名和类名问题

上述 + 标记的这一行中，新增了 public 关键字，表示这个是公共类。
如果某个类是 public ，那么这个类必须与源文件名一致。
一个源文件只能有一个 public 类。

一个源文件可以编写多个类吗？如果可以，编译后会怎么样呢？
如果一个源文件有多个类，那么编译后会产生多个.class字节码文件，且字节码文件名就是类名。

平时开发中，一个.java文件，只写一个类，除非有内部类，且源文件名和类名一致，因为后期好维护。
