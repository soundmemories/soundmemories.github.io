Java语言简介和环境搭建

发表于 2020-01-20 分类于 Java 本文字数： 1.9k 阅读时长 ≈ 7 分钟

Java语言

Java是一门面向对象编程语言，不仅吸收了 C++ 语言的各种优点，还摒弃了 C++ 里难以理解的多继承、指针等概念，因此Java语言具有功能强大和简单易用两个特征。
主要用于面向对象编程，Java语言适用于JavaEE、大数据开发、Android开发等领域。

Java概述

Java语言发展史

Java语言诞生于SUN（Stanford University Network）公司，最初命名为Oak（橡树）。
研发Java的小组命名为 Green Team ，小组成员有 James Gosling、Bill Joy、Patrick Naughton、Mike Sheridan。
最初的目的是跨平台与家电一起使用，直到1994年，小组意识到Oak非常适用于互联网。

1995年5月23日发布定名为Java。
1996年1月，第一个JDK-JDK1.0正式版本诞生。
1997年2月18日，JDK1.1发布。
1999年6月，SUN公司发布Java的三个版本：标准版（J2SE）、企业版（J2EE）和微型版（J2ME）。
2009年4月20日，Oracle公司以现金收购Sun微系统公司，交易价格达74亿美元，其主要目标是MySQL。
2014年3月18日，Oracle公司发表Java SE 8(LTS)。一共有10大新特性，包括Lambda表达式、Stream API、Data/Time API、函数接口、默认方法和静态方法等。
2017年9月21日，Oracle公司发表Java SE 9，尝试把JavaSE、JavaEE、JavaME合起来，用模块的方式进行区分。
2018年3月21日，Oracle公司发表Java SE 10。
2018年9月25日，Oracle公司发表Java SE 11 (LTS)。
2019年3月17日，Oracle公司发表Java SE 12。

目前大多数公司使用Java SE 8。

Java语言特征

  • Java语言是强制 面向对象的。 Java语言提供类、接口、继承等原语，为简单起见，只支持类之间的单继承，但支持接口之间的多继承、并支持类与接口之间的实现机制（关键字为implements）。
  • Java语言是分布式的。 Java语言支持Internet应用的开发，在基本的Java应用编程接口中有一个网络应用接口（java net），它提供了用于网络应用编程的类库，包括URL、URLConnection、Socket、ServerSocket等。Java的RMI（远程方法激活）机制也是开发分布式应用的重要手段。
  • Java语言是健壮的。 Java的强类型机制、异常处理、垃圾的自动收集等是Java程序健壮性的重要保证。对指针的丢弃是Java的明智选择。
  • Java语言是安全的。 Java通常被用在网络环境中，为此Java提供了一个安全机制以防止恶意代码的攻击。如：安全防范机制（类ClassLoader），如分配不同的名空间以防替代本地的同名类、字节代码检查。
  • Java语言是解释型的、体系结构中立的。 Java程序（.java的文件）在Java平台上被编译为体系结构中立的字节码格式（.class的文件），然后可以在实现这个Java平台的任何系统中运行。
  • Java语言是原生支持多线程的。 Java语言中，线程是一种特殊的对象，它必须由Thread类或其子（孙）类来创建。

Java语言特点

  • 特点一：面向对象
    • 两个基本概念：类、对象。
    • 三大特性：封装、继承、多态。
  • 特点二：健壮性
    • 吸收了 C/C++ 语言的优点，但去掉了影响程序健壮性的部分（如指针、内存的申请与释放等），提供了一个相对安全的内存管理和访问机制。
  • 特点三：跨平台
    • Java语言编写的应用程序在不同的系统平台上都可以运行，只要在需要运行Java应用程序的操作系统上，先安装一个Java虚拟机（JVM）即可，由JVM来负责Java程序在该系统中的运行。

Java两种核心机制

  • Java虚拟机（Java Virtal Machine）
    • JVM是一个虚拟计算机，具有指令集并使用不同的存储区域。负责执行指令，管理数据、内存、寄存器。对于不同的平台，有不同的虚拟机。
    • Java虚拟机机制屏蔽了底层运行平台的差别。
  • 垃圾回收机制（Garbage Collection）
    • 不再使用的内存空间应回收。在 C/C++ 语言中，由程序员负责回收无用内存，而Java提供一种系统级线程跟踪存储空间分配情况，并在JVM空闲时，检查并释放那些可被释放的存储空间。
    • 垃圾回收在Java程序运行中自动进行，有其自己的一套算法，程序员无法精确控制和干预。

开发环境搭建

JRE、JDK介绍

  • JRE（Java Runtime Environment）Java运行环境
    • 包括Java虚拟机（JVM）和Java程序所需的核心类库等，如果想运行一个开发好的Java程序，计算机中只需要安装JRE即可。
    • JRE = JVM + JavaSE标准类库。
  • JDK（Java Development Kit）Java开发工具包
    • JDK是提供给Java开发人员使用的，其中包含了Java的开发工具，也包括了JRE。所以安装了JDK，就不用单独安装JRE了。Java开发工具：编译工具（java.exe）+ 打包工具（jar.exe）等。
    • 简言之，使用JDK的开发工具完成的Java程序，交给JRE去运行。
    • JDK = JRE + 开发工具集（如javac编译工具等）

下载与安装

  • 下载：进入Oracle官网-> Products -> Software -> Java -> 页面右侧Download Java -> Java SE 8 -> Oracle JDK -> JDK Download -> 选择适合自己的下载即可。
  • 安装：安装目录不要有中文，然后一直next。
    • 安装可选项中开发工具和公共JRE都会安装一个JRE，不想多安装一个JRE可以取消公共JRE的安装。
    • 建议每一个软件的安装建立自己的目录，方便记住安装的位置，避免环境冲突何和覆盖。

环境变量

  • 为什么配置环境变量？
    • Windows等操作系统，默认是在当前路径下找“命令“等工具的。如果希望它在指定的目录下去查找，就需要在path这个环境变量中，把目录添加上去。
    • Windows等操作系统，除了在当前目录下查找，还会去path环境变量指定的目录下查找。
  • Windows的path在哪？
    • 计算机->右键->属性->高级系统设置->高级->环境变量。
  • 用户环境变量和系统环境变量区别？
    • 用户环境变量：只针对当前操作系统登陆的用户。
    • 系统环境变量：针对所有用户。
  • 如何添加环境变量？
    • 在系统环境变量中选择Path，在里面新建一条，把软件安装目录添加进去，如E:....。
    • Windows7对path添加时尾部加上 ; ，如E:....;。
