Java设计模式

发表于 2020-03-17 分类于 Java
本文字数： 12k 阅读时长 ≈ 11 分钟

设计模式

说白了，就是解决问题的套路，代码结构。
Java中常用的设计模式有23种。

JavaSE阶段给大家普及几个：
（1）模板设计模式（做到能模仿（3星））
（2）单例设计模式（做到脱稿能写（5星））
（3）工厂设计模式（做到会调用工厂类的方法（2星））
（4）代理设计模式（做到能模仿（3星））
（5）迭代器设计模式（了解）
（6）装饰者设计模式（了解）
其他复杂设计模式，做到能看懂代码，能认识。

模板设计模式（抽象类）

当你解决一下问题时，总体的算法的结构/步骤是确定的，但是其中的一步或几步的代码的具体实现是不确定的，需使用者自己来确定。

举例：编写一个类，包含一个方法，可以统计你执行任意代码的运行时间（时间的单位：毫秒）。
步骤：
（1）获取开始时系统时间；
（2）执行xxxx；
（3）获取结束时系统时间；
（4）计算时间差。
提示：System.currentTimeMillis()

单例设计模式（多线程）

建议学完Java多线程及前面的知识再看。

单例：某个类只能有唯一的一个实例对象。
比如，枚举中只有一个对象，就是最简单的单例。

如何实现单例？
（1）饿/恶汉式：不管是否需要这个对象，它都上来先给你创建好这个唯一的对象。参考TestSingle1。
形式一：1.5之后，用enum直接创建。
形式二：构造器私有化->用一个全局的静态的常量，来保存这个唯一的实例对象。
形式三：构造器私有化->用一个私有的静态的常量，来保存这个唯一的实例对象->提供一个静态方法，来返回这个常量对象。

（2）懒汉式：延迟创建对象。当使用者来或者这个对象，要用到对象时，我再创建。参考TestSingle2和TestSingle3。
形式一：构造器私有化->创建一个私有的静态的常量，但此时不创建唯一的实例对象给它->提供一个静态方法，当使用这个静态方法时，来创建实例对象给私有的静态的常量，然后返回这个常量对象；或之前创建过实例对象给私有的静态的常量，此时直接返回这个常量对象。
形式二：内部类形式。

  • 线程安全问题：多个线程时，可能几个线程都能判断没有实例对象，需要新建，此时返回的对象就不是同一个了。解决办法：把这个方法锁上，但性能较差。
  • 性能问题：安全问题只在第一次新建出现，如果把这个方法锁上，之后每次线程使用都要等锁，性能较差。解决办法：使用“当前类的Class对象”作为锁，外面再加一层判断有没有实例对象。外层的判断就是优化性能。

工厂设计模式（反射）

Java中为了解耦合，把对象的创建者与对象的使用者分开，批量生产对象。
在Java反射中，可知要用反射创建一个对象，代码可能非常复杂，此时就可以使用工厂设计模式来批量创建对象。

1、简单工厂模式
使用工厂类，可以new相应类的对象，使用者和这个对象之间没有直接关联，解耦合。

优点：代码比较简洁。
缺点：如果增加新的产品类型，需要修改工厂类，违反了面向对象的一个开发原则：对扩展开放，对修改关闭。


2、工厂方法模式
一个产品就对应一个工厂类，如果增加新产品，就不需要修改原来的工厂类。

优点：遵循了增加新产品，不修改原来的类的原则。
缺点：类太多了。

而使用了反射，更灵活，只需要更改配置文件即可。

代理设计模式

（1）当前这个类做一些事情（多变，重复的），不太方便，那么可以交给代理。
（2）当前这个类没有要用的资源，那么可以交给代理。

代理模式要求：
（1）主题接口：要求代理类与被代理类实现同一个接口。
（2）代理者：必须持有被代理者的引用。
（3）被代理者：核心类。

1、静态代理模式
缺点：一个代理类只能替一个代理主题（接口）代理工作。

2、动态代理模式
（1）编写一个代理工作处理器的类，这个类必须实现一个接口InvocationHandler。
（2）用JDK中提供了一个Proxy类，来创建代理类的对象。
（3）调用方法。
优点：一个代理工作处理器，可以替多个代理主题代理工作，只有代理工作内容一样就可以。

比如，测试类代码，就可以使用代理，方便后期上线。动态代理在Spring中很常见。
