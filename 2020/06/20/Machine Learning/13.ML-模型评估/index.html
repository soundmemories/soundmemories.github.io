ML-模型评估

发表于 2020-06-20 分类于 Machine Learning
本文字数： 15k 阅读时长 ≈ 13 分钟

常用词

错误率 ：分类错误的样本数占总样本数的比例。
精度：1-错误率。

训练误差/经验误差 ：机器学习器在训练集上的误差。
泛化误差 ：机器学习器在新样本上的误差。

过拟合：我们希望学习器在新样本下也能有好的表现，因此应该从训练样本中尽可能学出适用于所有潜在潜在样本的“普遍规律”，然而有时候学习器把训练样本学得“太好了”，很可能把训练样本自身的一些特点当作了所有潜在样本都具有的一般性质，这样就会导致泛化能力下降，这就称为过拟合。
欠拟合：和过拟合相对，即对训练样本的一般性质尚未学好。

生成模型与判别模型：
生成模型：学习得到联合概率分布P(x,y)P(x,y)P(x,y)，即特征x和标记y共同出现的概率，然后求条件概率分布。能够学习到数据生成的机制。通过求输入与输出的联合概率分布，再求解类别归类的概率。比如，朴素贝叶斯生成的模型。生成模型，对数据要求较高（比如朴素贝叶斯要求数据是离散的），速度会快些。
判别模型：学习得到条件概率分布P(y∣x)P(y|x)P(y∣x)，即在特征x出现的情况下标记y出现的概率。直接获得输出对应最大分类的概率。比如，KNN。判别模型，要求较低，有很好的容忍度，但速度会慢些。

数据划分

训练集：训练和拟合模型。提高训练集是好的，但比例不能变。
验证集：当通过训练集训练出多个模型后，使用验证集纠偏或比较预测。
测试集：模型的泛化能力考量。
测试集应该尽可能与训练集互斥，即测试样本尽量不在训练集中出现，未在训练过程中使用过。

一般情况下：训练集:测试集:验证集=6：2：2。
有时候会忽略验证集，而通过 训练集-测试集 不断的尝试来达到目的，此时训练集、测试集比例一般为8：2。

留出法（hold-out）

直接将数据集划分为两个互斥集合，一个作为训练，一个作为测试。

注意点：

  1. 保持数据分布一致性 ：如保留类别比例一致的分层采样。
  2. 多次重复划分 ：单次使用留出法得到的估计结果往往不够稳定可靠，一般要采用若干次随机划分、重复进行实验评估后取平均值作为留出法的结果。
  3. 训练集和测试集比例问题：在划分时，如果训练集包含绝大多数样本，则训练出来的模型更接近于未划分的数据集训练出的模型，由于测试集比较小，评估的结果可能不够准确；若测试集包含的样本多些，则训练集和未划分的数据集差别更大，这样训练出的模型偏离初衷，而且降低评估结果的保真性。常见的作法是2/34/5的样本用于训练，1/31/5的样本用于测试。

K折交叉验证（k-cross validation）

每个子集DiD_iDi​都尽可能的保持数据分布的一致性，即从DDD中通过分层采样得到。
最终返回的是这k个测试结果的均值，所以交叉验证评估结果的稳定性和保真性很大程度取决于k的取值。

常用的k值为5，10。
为减少因样本划分不同而引入的差别，kkk折交叉验证通常要随机使用不同的划分重复ppp次，最终的评估结果是这ppp次kkk折交叉验证结果的均值，例如常见的有10次10折交叉验证。

当数据集DDD中包含mmm个样本，若令k=mk=mk=m，则得到了交叉验证法的一个特例，留一法（Leave-One-Out）。
即mmm个样本只有唯一的方式划分为m个子集——每个子集包含一个样本；留一法使用的训练集与初始数据集相比只少了一个样本，这就使得在绝大多数情况下，留一法被实际评估的模型与期望评估的用DDD训练出的模型很相似，因此留一法的评估结果往往被认为比较准确。
然而，留一法也有其缺陷：在数据集比较大的时候，计算开销非常大。

自助法（boostrapping）

不管是留出法还是交叉验证法，都保留了一部分样本用于测试，因此实际评估的模型所使用的训练集比原始数据集DDD小，这必然会引入一些因训练样本规模不同而导致的估计偏差。留一法受训练样本规模变化的影响较小，但计算复杂度太高。有没有办法可以减少训练样本规模不同造成的影响，同时还能比较高效地进行实验估计呢？————自助法（boostrapping）。

自助法（boostrapping）直接以自助采样法为基础。给定包含mmm个样本的数据集DDD，对它进行采样产生数据集D′D^\primeD′：每次随机从DDD中挑选一个样本，将其拷贝放入D′D^\primeD′，然后再将该样本放回初始数据集DDD中，使得该样本在下次采样时仍可能被采到；这个过程重复执行mmm次后，我们就得到了包含mmm个样本的数据集D′D^\primeD′（和原数据集DDD同规模），这就是自助采样的结果，显然DDD中有一部分样本会在D′D^\primeD′中多次出现，而另一部分样本不出现，可以做一个简单估计，样本mmm次采样中始种不被采到的概率是(1−1m)m(1-\dfrac{1}{m})^{m}(1−m1​)m，取极限得到：lim⁡m→∞(1−1m)m→1e≈0.368\lim\limits_{m \to \infty}(1-\dfrac{1}{m})^{m} \to \dfrac{1}{e} \approx 0.368m→∞lim​(1−m1​)m→e1​≈0.368，即初始数据集DDD中约有36.8%36.8\%36.8% 的样本未出现在采样数据集D′D^\primeD′中，可以使用这些样本作为测试集（约为总量1/3），D′D^\primeD′作为训练集，这样的测试结果亦称为包外估计（out-of-bagestimate）

优点：自助法在数据集较小、难以有效划分训练/测试集时很有效。
缺点：自助法产生的数据集改变了初始数据集的分布，这会引入估计偏差，因此在数据量足够时，使用留出法/交叉验证更好

主要为：分类模型、回归模型、聚类模型、关联模型的评估。

准确率（Accuracy）：TP+TNTP+FP+FN+TN\dfrac{TP+TN}{TP+FP+FN+TN}TP+FP+FN+TNTP+TN​，注意只用准确率不行，比如样本极度不平衡标签0：1的比例为99：1，而模型只要是输入就判别为0，准确率还是能达到99%，所以只用准确率判别是不好的。

精确率（Precision）：P=TPTP+FPP=\dfrac{TP}{TP+FP}P=TP+FPTP​，预测结果正类中，正确的程度。

召回率（Recall/TPR）：R=TPTP+FNR=\dfrac{TP}{TP+FN}R=TP+FNTP​，真实情况正类中，预测出的占比。

FβF_\betaFβ​-score：(1+β2)∗Precision∗Recallβ2∗Precision+Recall\dfrac{(1+\beta^2)*Precision*Recall}{\beta^2*Precision+Recall}β2∗Precision+Recall(1+β2)∗Precision∗Recall​，F1的一般形式，β>1\beta >1β>1时召回率有更大的影响；β<1\beta<1β<1时精确率有更大影响。

错误接收率（False Positive Rate/FPR）：FPFP+TN\dfrac{FP}{FP+TN}FP+TNFP​，真实负类中，被预测为正类的比例。

错误拒绝率（False Rejection Rate/FRR）：FNTP+FN\dfrac{FN}{TP+FN}TP+FNFN​，真实正类中，被预测为负类的比例。

多元混淆矩阵

在多次训练/测试后会有多个二分类混淆矩阵，或者在多个数据集上训练/测试后希望估计算法全局性能，或者执行多分类任务，每两两类别的组合都能对应一个混淆矩阵，总之我们希望在n个二分类混淆矩阵上综合考察精确率和召回率。

一种直接的做法是先在各混淆矩阵上分别计算出精确率和召回率，记为(P1,R1),(P2,R2),...,(Pn,Rn)(P_1,R_1),(P_2,R_2),...,(P_n,R_n)(P1​,R1​),(P2​,R2​),...,(Pn​,Rn​)，再计算平均值，这样就得到宏精确率（macro-P）和宏召回率（macro-R），以及相应的宏F1（macro-F1）。
macro-P：1n∑i=1NPi\dfrac{1}{n}\sum\limits_{i=1}^NP_in1​i=1∑N​Pi​

macro-R：1n∑i=1NRi\dfrac{1}{n}\sum\limits_{i=1}^NR_in1​i=1∑N​Ri​

macro-F1：2×macro-P×macro-Rmacro-P+macro-R\dfrac{2\times \text{macro-P}\times\text{macro-R}}{\text{macro-P}+\text{macro-R}}macro-P+macro-R2×macro-P×macro-R​

另一种是，先将混淆矩阵的对应元素进行平均，得到TPTPTP、FPFPFP、TNTNTN、FNFNFN的平均值，分别记为TP‾\overline{TP}TP、FP‾\overline{FP}FP、TN‾\overline{TN}TN、FN‾\overline{FN}FN，再基于这些平均值计算出微精确率（micro-P）和微召回率（micro-R），以及相应的微F1（micro-F1）。
macro-P：TP‾TP‾+FP‾\dfrac{\overline{TP}}{\overline{TP}+\overline{FP}}TP+FPTP​

macro-R：TP‾TP‾+FN‾\dfrac{\overline{TP}}{\overline{TP}+\overline{FN}}TP+FNTP​

macro-F1：2×macro-P×macro-Rmacro-P+macro-R\dfrac{2\times \text{macro-P}\times\text{macro-R}}{\text{macro-P}+\text{macro-R}}macro-P+macro-R2×macro-P×macro-R​

PR曲线

根据学习器的预测结果，对样例进行排序，排在最前面的是学习器认为最可能是正例的样本，排在最后面的是学习器认为最不可能是正例的样本。按此顺序逐个把样本作为正例进行预测，则每次可以计算出当前的召回率（查全率）、精确率（查准率）。以精确率为纵轴，召回率为横轴，作图就得到了精确率-召回率曲线，简称P-R曲线。

P-R曲线能直观的显示出学习器在样本总体上的精确率和、召回率。在比较时，若一个学习器的P-R曲线被另一个学习器的曲线完全包住，则可断言后者性能优于前者。

ROC曲线与AUC

ROC（Receiver Operating characteristic Curve）根据学习器的结果对样例进行排序，按此顺序逐个把样本作为正例进行预测，每次可以计算出当前的召回率TPR和错误接收率FPR，作为纵轴和横轴。我们希望TPR越大越好，FPR越小越好。ROC曲线能很容易的查出任意阈值时，对性能的识别能力。一般我们取拐点处的阈值最佳，这兼顾了TPR和FPR。

TPR：TPTP+FN\dfrac{TP}{TP+FN}TP+FNTP​
FPR：FPTN+FP\dfrac{FP}{TN+FP}TN+FPFP​

若一个学习器的ROC曲线被另一个学习器的曲线完全包住，则可断言后者性能优于前者。若两个学习器的ROC曲线发生交叉，则难以断定孰优孰劣，此时看AUC面积。
AUC（Area Under Curve）就是ROC曲线下方的面积，它反映了曲线向左上方靠近程度。越大越好。

ROC和PR区别：

An introduction to ROC analysis
根据作者原文来看，负例增加了10倍，ROC曲线没有改变，而PR曲线则变了很多。作者认为这是ROC曲线的优点，即具有鲁棒性，在类别分布发生明显改变的情况下依然能客观地识别出较好的分类器。
不过在类别不平衡的背景下，负例的数目众多致使FPR的增长不明显，导致ROC曲线呈现一个过分的效果估计。

ROC和PR应用场景：
ROC曲线由于兼容正例与负例，所以适合评估分类器的整体性能，相比而言PR曲线完全聚焦于正例。

如果有多份数据且存在不同的类别分布，比如信用卡欺诈问题中，每个月正例和负例的比例可能都不相同，这时候如果只想单纯地比较分类器的性能且剔除类别分布改变的影响，则ROC曲线比较适合，因为类别分布的改变可能使得PR曲线发生变化时好时坏，这种时候难以进行模型比较。反之，如果想测试不同类别分布下，对分类器的性能的影响，则PR曲线比较合适。

如果想要评估在相同的类别分布下正例的预测情况，则宜选PR曲线。

类别不平衡问题中，ROC曲线通常会给一个乐观的效果估计，所以大部分时候还是PR曲线更好。

增益图与KS图

左图是增益图，虚线是1。比如挖取游戏作弊玩家，100个玩家，10个作弊的：计算得分，取出得分前10的用户，发现有9个作弊的，那么正样本比例是9/10=0.9，测试集取样比例为10/100=0.1，正样本比例/平均比例为0.9/0.1=9。增益图最好情况是在取样比例为0.5为转折点，前部分全是最高值，后部分全是最低值。增益图宏观上反映了分类器的分类效果。

右图是KS图，我们关心的是两条曲线的最大差值，横坐标是阈值，纵坐标是差值TPR−FPRTPR-FPRTPR−FPR，这个差值反映了对正类样本的区分度。一般选择差值最大的阈值作为分类阈值。

MSE（Mean Square Error）均方误差: 1n∑i=1n(fi−yi)2=1n∑i=1n(ei)2\dfrac{1}{n}\sum\limits_{i=1}^n (f_i - y_i)^2=\dfrac{1}{n}\sum\limits_{i=1}^n(e_i)^2n1​i=1∑n​(fi​−yi​)2=n1​i=1∑n​(ei​)2，真实值和预测值差值的平方和。

RMSE（Root MSE）均方根误差: MSE\sqrt{MSE}MSE​，对MSE开根号，如果MSE都是比较小的，可以放大它们之间尺度。

r2_score（决定系数）: R2=∑i=1n(yi−fi)2∑i=1n(yi−yˉ)2R^2=\dfrac{\sum\limits_{i=1}^n ( y_i - f_i)^2}{\sum\limits_{i=1}^n ( y_i - \bar y)^2}R2=i=1∑n​(yi​−yˉ​)2i=1∑n​(yi​−fi​)2​，fif_ifi​是预测值，yiy_iyi​是实际值，yˉ\bar yyˉ​是实际值的平均值。分子代表预测值对实际值的离散程度，分母代表真实值的离散程度。R平方为回归平方和与总离差平方和的比值，表示总离差平方和中可以由回归平方和解释的比例，这一比例越大越好，模型越精确，回归效果越显著。R平方介于0~1之间，越接近1，回归拟合效果越好，一般认为超过0.8的模型拟合优度比较高。

聚类模型

RMS（Root Mean Square）： 1n∑i=1n(xi−xˉ)2\dfrac{1}{n}\sqrt{\sum\limits_{i=1}^n (x_i-\bar x)^2}n1​i=1∑n​(xi​−xˉ)2​，聚类的值减去每个类的平均值，然后平方和开根号，除以n。越小越好，越大表示每个类和它的中心比较远，效果差。
