ML-IPython和Jupyter

发表于 2020-04-21 分类于 ML 本文字数： 4.5k 阅读时长 ≈ 16 分钟

IPython

IPython（interactive Python 的简称，即交互式Python），由 Fernando Perez 作为一个增强的 Python 解释器于 2001 年启动，并由此发展为一个项目。用 Perez 的原话来说，该项目致力于提供“科学计算的全生命周期开发工具”。如果将 Python 看作数据科学任务的引擎，那么 IPython 就是一个交互式控制面板。

IPython 除了作为 Python 的一个交互式接口，还提供了一些有用的 Python 语法附加功能（魔法命令），我会介绍其中最有用的一些。

另外，IPython 被紧密地连接在 Jupyter项目 中。该项目提供一个基于浏览器的 Notebook，它可以开发、协作、分享甚至发布数据科学结果。IPython Notebook 其实只是通用 Jupyter Notebook 结构的特例，而Jupyter Notebook 不仅支持 Python，还包括用于 Julia、 R 和其他编程语言的 Notebook。

Jupyter

Jupyter Notebook 是 IPython shell 基于浏览器的图形界面，提供了一系列丰富的动态展示功能。 Jupyter Notebook 不仅可以执行 Python/IPython 语句，还允许用户添加格式化文本、静态和动态的可视化图像、数学公式、 JavaScript 插件，等等。不仅如此，这些 Notebook 文档还能以共享方式存储，以便其他人可以打开这些 Notebook，并且在他们自己的系统中执行这些 Notebook 代码。

IPython Notebook 是通过你的 Web 浏览器窗口进行查看和编辑的，但是它必须与一个正在运行的 Python 进程连接才能执行代码。想要启动这个进程（也被称作“核”， kernel），需要在你系统的命令行cmd中输入以下命令：

IPython魔法命令

IPython 在普通 Python 语法基础之上的增强功能，被称作 IPython 魔法命令，并且都以 % 符号作为前缀。
这些魔法命令设计用于简洁地解决标准数据分析中的各种常见问题。

魔法命令有两种形式：
（1）行魔法（line magic）：以单个 % 字符作为前缀，作用于单行输入。
（2）单元魔法（cell magic）：以两个 %% 作为前缀，作用于多行输入。

输入和输出对象

IPython 用到的 In[1]:/Out[1]: 形式，对应输入和输出对象，数字代表按照顺序执行的命令数。
In 对象是一个列表，按照顺序记录所有的命令。
Out 对象是一个字典。它将输入数字映射到相应的输出。


不是所有操作都有输出，任何返回值是 None 的命令都不会加到 Out 变量中，例如 import 语句和 print 语句就不影响输出。

获取输出历史：Out[X] 的简写形式是 X，变量 （单下划线）代表上次的输出：

用两条下划线获得倒数第二个历史输出，用三条下划线获得倒数第三个历史输出，以此类推。
超过三条下划线开始变得比较难计数，并且在这种情况下通过行号 Out[行号] 来指定输出更方便。

获取输入历史：%history 可一次性获取此前所有的输入历史。

默认情况下，这个轨迹信息显示了导致错误的每个步骤的上下文。利用 %xmode 魔法函数（简称异常模式），可以改变打印的信息。

默认情况下是 Context，该模式的输出结果上面已经见过。 Plain 更紧凑，给出的信息更少。Verbose 模式加入了一些额外的信息，包括任何被调用的函数的参数，如果代码复杂，这种方式的轨迹追溯会变得非常长。根据不同情境，有时默认模式的简要描述更容易处理。

当阅读轨迹追溯不足以解决问题时，使用交互式调试工具 pdb，它是 Python 的调试器。这个调试器允许用户逐行运行代码，以便查看可能导致错误的原因。 IPython 增强版本的调试器是 ipdb，它是 IPython专用的调试器。我简单的写了Python-pdb调试器介绍其使用方法。

在有pdb或ipdb的情况下，可以使用%debug，它会捕获异常后调用该调试器，在异常点自动打开一个交互式调试面板。

如果你希望发生任何异常时都自动启动调试器，可以用 %pdb on 魔法函数。

时间分析

%timeit 它会自动计算接下来一行的 Python 语句的执行时间。
%%timeit 它会自动计算接下来一单元的 Python 语句的执行时间。


对于多行语句，可以加入第二个 % 符号将其转变成单元魔法，以处理多行输入。或者开启nbextensions中的Excute Time。

%timeit 它会自动多次重复执行命令，以获得更稳定的结果。有时候重复一个操作并不是最佳选择，例如，如果有一个随机列表需要排序，这可能会被重复操作误导。对于这种情况， %time 魔法函数可能是更好的选择，同样 %%time 单元魔法。

%timeit 通常比 %time 更快得到结果。
