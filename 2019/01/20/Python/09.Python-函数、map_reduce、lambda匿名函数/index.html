Python-函数、map/reduce、lambda匿名函数

发表于 2019-01-20 分类于 Python
本文字数： 9.9k 阅读时长 ≈ 9 分钟

函数定义、调用

函数：可多个输入，但输出是一个

  1. 函数名是标识符，命名要求一样
  2. 语句块必须缩进，约定4个空格
  3. Python的函数没有return语句，隐式会返回一个None值
  4. 定义中的参数列表称为形式参数，只是一种符号表达式（表示符），简称形参 调用：
    函数定义，只是声明了一个函数，它不会被执行，需要调用的方式，就是函数名后加上小括号，如有必要在括号内写上参数；调用时写的参数是实际参数，是实际传入的值，简称实参。
  5. 上面只一个add函数的定义，函数名叫add，它接受2个参数；
  6. add函数计算的结果通过return语句返回；
  7. 调用：函数名add加2个参数，比如：add(4,5)，返回值可使用变量接收，比如：result = add(4,5)。函数名也是标识符，返回值也是值；
  8. 定义需要在调用前；也就是说调用时，它已经被定义过了，否则抛NameError异常；
  9. 函数是可调用的对象，callable(x)->bool：判断x是否是可调用对象；

传参传递

参数调用时传入的参数要和定义的个数相匹配（可变参数例外）。

关键字传参（python适用）、位置传参：

  1. 位置传参：
  • 按照参数定义顺序传入实参；

    举例：def f(x, y, z) 调用使用 f(1, 3, 5)；

  1. 关键字传参：
  • 使用形参的名字来传入实参的方式，如果使用了形参名字，那么传参顺序就可和定义顺序不同；

    举例：def f(x, y, z) 调用使用 f(x=1, y=3, z=5)；

  1. 传参：
  • 要求位置参数必须在关键字参数之前传入，位置参数是按位置对应的；

    举例：f(z=None, y=10, x=[1]) 正确；
    举例：f((1,), z=6, y=4.1) 正确；
    举例：f(y=5, z=6, 2) 错误；

    缺省参数

  • 默认值就是缺省值，在函数定义时可以在形参后添加一个缺省值；
  • 参数的默认值可以在未传入足够的实参的时候，对没有给定的参数赋值为默认值；
  • 参数非常多的时候，并不需要用户每次都输入所有的参数，简化函数调用；

    可变参数

    位置可变参数：
  • 一个形参可以匹配任意个参数（可变是0到任意个）；
  • 在形参前使用 ”*“ 表示该形参是可变参数，可以接收多个实参！
  • 收集多个实参为一个tuple！
    思考：关键字参数能否也能传递任意多个吗？可以

关键字可变参数：

  • 形参前使用 “**” 符号，表示可以接收多个关键字参数！
  • 收集的实参名称和值组成一个字典！

keyword-only参数（命名关键字参数）：

  • 如果在一个 “*” 参数后，或者一个位置可变参数后，出现的普通参数，实际上已经不是普通的参数了，而是keyword-only参数。 思考：def fn(**kwargs, x) 可以吗？
    答：可以理解为kwargs会截获所有的关键字参数，就算你写了x=5，x也永远得不到这个值，所以语法错误。

总结：

  • 位置/关键字>位置>*关键字；
  • 有位置可变参数和关键字可变参数！
  • 位置可变参数在形参前使用一个星号*！
  • 关键字可变参数在形参前使用两个星号**！
  • 位置可变参数和关键字可变参数都可以收集若干个实参，位置可变参数收集形成一个tuple，关键字可变参数收集形成一个dict！
  • 混合使用参数的时候，普通参数需要放到参数列表前面，可变参数要放到参数列表的后面，位置可变参数需要在关键字可变参数之前！

参数规则：

  • 参数列表参数一般顺序是，普通参数、缺省参数、可变位置参数、keyword-only参数（可带缺省值）、可变关键字参数。

注意：代码应该易读易懂，而不是为难别人；请按照书写习惯定义函数参数。

参数解构

  • 给函数提供实参的时候，可以在集合类型前使用或者*，把集合类型的结构解开，提取出所有元素作为函数的实参！
  • 非字典类型使用*解构成位置参数！
  • 字典类型使用**解构成关键字参数！
  • 提取出来的元素数目要和参数的要求匹配，也要和参数的类型匹配！

    返回值

    Python函数使用return语句返回返回值：
  • 所有函数都有返回值，如果没有return语句，隐式调用return None；
  • 一个函数可以存在多个return语句，但是只有一条可以被执行。如果没有一条return语句被执行到，隐式调用return None！
  • 如果函数执行了return语句，函数就会返回，直接结束函数！
  • 如有必要，显示调用return None，可以简写为return！
  • return语句并不一定是函数的语句块的最后一条语句，可以在函数内任意位置添加return语句！

简单总结：

  1. 所有函数都有返回值，不写return隐式返回None；
  2. 只有一个return生效，后面即使还有return也不执行；
  3. 遇到return就直接结束退出；
  4. return返回多值时隐式封装成一个元组

作用域

（1）Local：在程序的最内层，包含局部变量，比如，一个函数的内部。函数调用时创建，调用结束消亡。
（2）Enclosing：包含了非局部(non-local)也非全局(non-global)的变量。比如两个嵌套函数，函数/类 A里面又包含了函数B，那么对于B中的名称来说A中的作用域就为no-nlocal。
（3）Global：当前脚本的最外层，比如，当前模块的全局变量。模块被import时创建，解释器退出时消亡。
（4）Built-in： 包含了内建的变量/关键字等，比如，int。 生命周期从python解释器启动时创建到解释器退出时消亡。
查找顺序：Local->Enclosing->Global->Build-in。

每一次函数调用，里面的变量都会重新生成一次。
函数调用就是压栈的过程。
外层作用域变量在内层作用域可见，内层可以使用不可修改。


如何在函数内部使用全局变量：global声明，注意会影响全局变量，不建议实际项目使用。

函数使用原则：
1.内部不要隐式使用外部全局变量；
2.内部函数应和外部完全隔离，只有传参接口；
3.不用global，学习它就是为了深入理解变量作用域；

自由变量：未在本地作用域中定义的变量。例如，定义在内层函数外的外层函数作用域中的变量。

缺省值为引用类型

当缺省值是引用类型时要多加注意：
（1）使用可变类型作为默认值，就可能修改这个默认值。
（2）有时候这个特性是好的，有的时候这种特性是不好的，有副作用。

属性__defaults__中使用元组保存所有位置参数默认值。
属性__kwdefaults__中使用字典保存所有keyword-only参数的默认值。

函数的销毁

全局函数销毁：
（1）重新定义同名函数，赋值即定义，覆盖掉原函数；
（2）del 删除函数名称，函数对象引用计数减1；
（3）程序结束时自动销毁（引用计数为零）；


局部函数销毁：
（1）重新在函数上级作用域定义同名函数；
（2）del,语句删除函数名称，函数对象的引用计数减1；
（3）上级作用域销毁时（引用计数为零）；

高阶函数

map/reduce

map()函数接收两个参数，一个是函数，一个是Iterable，map将传入的函数依次作用到序列的每个元素，并把结果作为新的Iterator返回。


reduce: 用传给 reduce 中的函数 function（有两个参数）先对集合中的第 1、2 个元素进行操作，得到的结果再与第三个数据用 function 函数运算，依此类推，最后得到一个结果。

sorted/reversed/zip

  • sorted(iterable[,key][reverse])：立即返回一个新列表！默认升序。可迭代对象就能排序。
  • reversed(seq)：返回一个反转的 迭代器，必须是有序类型。
  • zip(*iterables)：拉链函数，惰性，是迭代器。把n个迭代对象封装成m个n元组，m按迭代对象元素最少算(每个中取一个元素配对)。
  • enumerate(iterable,start=0)：返回一个迭代器。前面从start开始编号，后面是值的2元组。

    sort是应用在list上的方法，sorted可以对所有可迭代的对象进行排序操作。
    list的sort方法返回的是对已经存在的列表进行操作，而内建函数sorted方法返回的是一个新的list，而不是在原来的基础上进行的操作。

（1）使用lambda关键字来定义匿名函数；

递归函数

递归函数（主要是理解压栈原理）：
（1）函数直接或者间接调用自身就是递归；
（2）递归需要有边界条件、递归前进段、递归返回段；
（3）递归一定要有边界条件；
（4）当边界条件不满足的时候，递归前进；
（5）当边界条件满足的时候，递归返回；
慎用，效率低！
建议可视化观察程序压栈、出栈过程！pythontutor

上面执行过程：
（1）全局帧中生成foo1、 foo2、 foo3、 main函数对象；
（2）main函数调用：
（3）main中查找内建函数print压栈，将常量字符串压栈，调用函数，弹出栈顶；
（4）main中全局查找函数foo1压栈，将常量100、101压栈，调用函数foo1，创建栈帧。print函数压栈，字符串和变量b、b1压栈，调用函数，弹出栈顶，返回值；
（5）main中全局查找foo2函数压栈，将常量200压栈，调用foo2，创建栈帧。foo3函数压栈，变量c引用压栈，调用foo3，创建栈帧。foo3完成print函数调用后返回。foo2恢复调用，执行print后，返回值。main中foo2调用结束弹出栈顶。main继续执行print函数调用，弹出栈顶。main函数返回。

斐波那契数列是递归最好的例子，公式：f(n)=f(n-1)+f(n-2)

压栈：先算f(n-1)，把f(n-1)递归压栈直到结束，f(n-1)压栈计算全结束后再递归压栈f(n-2)。

总结：一般的递归性能太低，因为每次计算都要重新把前面的计算一遍，一般能用递归的也能用for循环实现，即使递归代码很简洁，但是能不用则不用递归；
