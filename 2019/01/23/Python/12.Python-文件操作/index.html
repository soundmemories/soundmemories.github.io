打开操作

打开一个文件，返回一个文件对象(流对象)和文件描述符。打开文件失败，则返回异常。

  • file：打开或者要创建的文件名。如果不指定路径，默认是当前路径。
  • mode：
    • ‘r’：缺省的，只读打开。
    • ‘w’：只写打开。file不存在时新建，file存在时清空内容后打开，每次打开都是空白文件，慎用！
    • ‘x’：新建file，file存在则报错，只写打开。
    • ‘a’：file内容末尾追加，只写打开，file不存在则新建。
    • ‘b’：二进制模式。
    • ‘t’：缺省的，文本模式。open的默认模式其实是’rt’。
    • ‘+’：给只读、只写方式追加缺失的读或者写模式。

      文本模式t：字符流，将文件的字节按照某种字符编码理解，按照字符操作。open的默认mode就是rt。
      二进制模式b：字节流，将文件就按照字节理解，与字符编码无关。二进制模式操作时，字节操作使用bytes类型。

windows中使用codepage代码页，可以认为每一个代码页就是一张编码表。cp936等同于GBK。

文件指针

文件指针，指向当前字节位置。
mode=r，指针起始在0。
mode=a，指针起始在EOF。


tell()：显示指针当前位置（字节为单位）。
seek(offset[, whence]：移动文件指针位置。offest偏移多少字节，whence从哪里开始。

  • whence（文本模式）：
    • 0：缺省值，表示从头开始，offest只能正整数。
    • 1：从当前位置，offest只接受0，相当于原地不动，所以没什么用。
    • 2：从EOF开始，只支持偏移0，相当于移动文件指针到EOF。
  • whence（二进制模式）：
    • 0：缺省值，表示从头开始，offest只能正整数。
    • 1：从当前位置，offest可正可负。
    • 2：从EOF开始，offest可正可负。

      二进制模式支持任意起点的偏移，从头、从尾、从中间位置开始。
      向后seek可以超界，但是向前seek的时候，不能超界，否则抛异常。

  • seek_b

buffering

  • buffering缓冲区：
    • -1：使用缺省大小的buffer。
      • b模式时，使用io.DEFAULT_BUFFER_SIZE值，默认是4096或者8192。
      • t模式时，若是终端设备则行缓存方式，若不是则使用b模式的策略。
    • 0：
      • b模式时，关buffer。
      • t模式不支持。
    • 1：
      • b模式时，一个字节缓冲区。
      • t模式时，行缓冲，见到‘\n’就flush全部缓冲区数据。
    • >1：用于指定buffer大小。
      • b模式时，表示缓冲大小。设定值可以超过io.DEFAULT_BUFFER_SIZE，超出设定值才把缓冲区flush。
      • t模式时，缓冲大小是io.DEFAULT_BUFFER_SIZE个字节（缺省缓冲区大小），flush完后把当前字符串也写入磁盘。

io.DEFAULT_BUFFER_SIZE：缺省缓冲区大小（字节）。
buffer缓冲区：缓冲区一个内存空间，一般来说是一个FIFO队列，到缓冲区满了或者达到阈值，数据才会flush到磁盘。
flush()将缓冲区数据写入磁盘。
close()关闭前会调用flush()。

  • buffering-t
  • buffering_b

文本模式，一般都用默认缓冲区大小。
二进制模式，字节操作，可以指定buffer的大小。
编程中，明确知道需要写磁盘了，都会手动调用一次flush，而不是等到自动flush或者close的时候。

其他参数

encoding：编码，仅限t模式。
encoding=None 表示使用缺省编码，依赖操作系统。
windows下缺省GBK(0xB0A1)，Linux下缺省Utf-8(0xE5 95 8A)。

errors：什么样的编码错误将被捕获。
errors=None/strict 表示有编码错误将抛出ValueError异常；ignore表示忽略。

newline：指定换行符。
t模式中，可以为None、’’、’\r’、’\n’、’\r\n’或其他合法字符。
read时，newline=None 会把’\r’、’\n’、’\r\n’转换为’\n’；’’ 不自动转换为’\n’；其它合法字符时，就会按照指这个字符分行。
write时，newline=None 会把’\n’替换为系统缺省行分隔符os.linesep；’\n’ 或’’ 表示’\n’不替换；其它合法字符时，’\n’会被替换成这个字符。

closefd：关闭文件描述符，True关闭，False会在文件关闭后保持这个描述符。fileobj.fileno()查看。

csv模块

CSV 是一个被行分隔符、列分隔符划分成行和列的文本文件。行称为记录，列称为字段。
CSV 不指定字符编码。
CSV 行分隔符为\r\n，最后一行可以没有换行符。
CSV 列分隔符常为逗号或者制表符。
CSV 字段可以使用双引号括起来，也可以不使用。如果字段中出现了双引号、逗号、换行符必须使用双引号括起来。如果字段的值是双引号，使用两个双引号表示一个转义。

读操作

返回reader对象，是一个行迭代器。
dialect：方言，缺省为excel方言。
delimiter：列分隔符，缺省为,逗号。
lineterminator：行分隔符，缺省为\r\n。
quotechar：字段的引用符号，缺省为"双引号。
doublequote：双引号的处理，缺省为True。如果碰到数据中有双引号，而quotechar也是双引号，True则使用2个双引号表示；False则使用escapecha作为双引号的前缀。
escapechar：一个转义字符，缺省为None。
quoting：指定双引号的规则。QUOTE_ALL（所有字段）；QUOTE_MINIMAL（特殊字符字段，Excel方言使用该规则）；QUOTE_NONNUMERIC（非数字字段）；QUOTE_NONE（都不使用引号）。

json模块

内存中的字典、列表、集合以及各种对象，如何保存到一个文件中？
如果是自己定义的类的实例，如何保存到一个文件中？
如何从文件中读取数据，并让它们在内存中再次恢复成自己对应的类的实例？

要设计一套协议，按照某种规则，把内存中数据保存到文件中。
把数据转换成字节序列，输出到文件，这就是序列化。
把文件的字节序列恢复到内存并且还是原来的类型，就是反序列化。

序列化后数据可持久化，或者网络传输；也可以将从文件中或者网络接收到的字节序列反序列化。
Python 提供了pickle 库。


Python程序之间还可以都是用pickle解决序列化、反序列化，如果是跨平台、跨语言、跨协议时，pickle就不适合了，就需要公共的协议。例如XML、Json、Protocol Buffer等。

JSON(JavaScript Object Notation, JS 对象标记) 是一种轻量级的数据交换格式。它基于 ECMAScript (w3c组织制定的JS规范)的一个子集，采用完全独立于编程语言的文本格式来存储和表示数据。
本质上来说JSON序列化后就是字符串。

json的值：string、number、object、array、true/false、null。
（1）string：由双引号包围起来的任意字符的组合，可以有转义字符。
（2）number：有正负，有整数、浮点数。
（3）object：无序的键值对的集合。 例如：{key1:value1, … ,keyn:valulen}，key必须是一个字符串，需要双引号包围这个字符串，value可以是任意合法的值。
（4）array：有序的值的集合。例如：[val1,…,valn]。
