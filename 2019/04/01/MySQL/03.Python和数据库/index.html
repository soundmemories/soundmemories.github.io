MySQL

驱动

MySQL基于TCP协议之上开发，但是网络连接后，传输的数据必须遵循MySQL的协议。
封装好MySQL协议的包，就是驱动程序。

MySQL的驱动：
（1）MySQLdb：对MySQL的C Client封装实现，支持Python 2，不更新了，不支持Python3。
（2）MySQL官方Connector：支持Python 3。
（3）pymysql：语法兼容MySQLdb，使用Python写的库，支持Python 3。

MySQL官方

由于MySQL服务器以独立的进程运行，并通过网络对外服务，所以，需要支持Python的MySQL驱动来连接到MySQL服务器。

pymysql

安装pymysql：

首先，必须建立一个传输数据通道——连接。
pymysql.connect()：返回Connection类实例，建立一个连接实例。常用参数：host，user，password，database，port。


连接实例提供方法：
ping()：可以测试数据库服务器是否活着。有一个参数reconnect表示断开与服务器连接是否重连。连接关闭抛出异常。
cursor(cursor=None)：返回一个游标对象。cursor可指定Cursor类，默认None是Cursor类。连接没有关闭前，游标对象可以反复使用。操作数据库，必须使用游标，所以必须创建游标对象。
begin()：开始事务。
commit()：将变更提交。
rollback()：回滚事务。

游标实例提供方法：
execute()：执行SQL语句，成功返回影响的行数。注意自动提交开启情况，默认没开启，需手动处理事务提交。
executemany()：批量执行，支持参数化。
fetchone()：该方法获取结果集的第一行。结果集是一个对象。
fetchmany(size=None)：指定获取结果集的size行，None则返回空元组。
fetchall()：获取结果集的所有行。返回多行，如果走到末尾，就返回空元组，否则返回一个元组，其元素就是每一行的记录。每一行的记录也封装在一个元组中。
rownumber：属性，返回当前行号。可以修改，支持负数。
rowcount： 只读属性，并返回执行execute()方法后影响的行数。

注意：fetch操作的是结果集，结果集是保存在客户端的，也就是说fetch的时候，查询已经结束了。

这就是ORM技术：Object-Relational Mapping，把关系数据库的表结构映射到对象上。ORM框架做的就是这个转换。

在Python中，最有名的ORM框架是SQLAlchemy，使用面向对象的方式来操作数据库。
SQLAlchemy做的转换：table->class、row->object、column->property。其中类属性是表的属性，实例属性是行的值。

常用的方法（实体是persistent状态）：
add()：增加一个对象。
add_all()：可迭代对象，元素是对象。
query()：查询，返回一个可迭代的Query对象，只有迭代它时才执行SQL来查询数据库。其后可以接filter、get、聚合函数等方法。

每一个记录，都有一个状态属性_sa_instance_state，其类型是sqlalchemy.orm.state.InstanceState，可以使用sqlalchemy.inspect(entity)查看状态。

常见的状态：
（1）transient：实体类尚未加入到session中，同时并没有保存到数据库中。
（2）pending：transient的实体被add()到session中，状态切换到pending，但它还没有flush到数据库中。
（3）persistent：pending状态在commit成功后变成persistent状态，或者查询成功返回的实体也是persistent状态。
（4）deleted：实体被删除，且已经flush但未commit完成。事务提交成功了，实体变成detached，事务失败，返回persistent状态。
（5）detached：删除成功的实体进入这个状态。

新建一个实体，状态是transient。
一旦add()后从transient变成pending状态。
成功commit()后从pending变成persistent状态。
成功查询返回的实体对象，也是persistent状态。
persistent状态的实体，修改后依然是persistent状态。
persistent状态的实体，删除+flush后，没有commit，就变成deteled状态，成功提交，变为detached状态，
提交失败，还原到persistent状态。flush方法，主动把改变应用到数据库中去。

删除、修改操作，需要对应一个真实的记录，所以要求实体对象是persistent状态。


其他的方法：
all()：返回列表，查不到返回空列表。
first()：返回首行，查不到返回None，等价limit。
one()：提取结果。使用one前结果必须是1个，否则报错。
scalar()：取one()返回元组的第一个元素。
group_by()：分组。
join()：关联查询。
