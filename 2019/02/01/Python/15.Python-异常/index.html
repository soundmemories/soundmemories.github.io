Python-异常

发表于 2019-02-01 分类于 Python 本文字数： 2.1k 阅读时长 ≈ 7 分钟

错误和异常

逻辑错误：算法写错了，如加法写成了减法。
语法错误：编译都过不去，函数或类使用错误。
总之，错误是可以避免的，应尽量避免错误。

异常：不是错误，是意外情况。程序上面没有问题，但在某些情况下出现意外，导致程序无法正常执行下去。
例如，open函数操作一个文件，文件不存在，产生异常。计算10/a，表达式是没有问题的，但当a是0时，产生异常。
总之，异常是不可避免的，应尽可能的捕获、处理各种异常。

BaseException

BaseException及子类

BaseException：所有内建异常类的基类。
KeyboardInterrupt：对应的捕获用户中断行为Ctrl + C。
SystemExit：sys.exit()函数引发的异常，异常不捕获处理，就直接交给Python解释器，解释器退出。


如果except语句捕获了该异常，则继续向后面执行，如果没有捕获住该异常SystemExit，解释器直接退出程序。

Exception及子类

Exception：是所有内建的、非系统退出的异常的基类，自定义异常类应该继承自它。
SyntaxError：语法错误，但是这种错误是不可捕获的。
ArithmeticError：所有算术计算引发的异常，其子类有除零异常等。
LookupError：使用映射的键或序列的索引无效时引发的异常的基类：IndexError, KeyError。

异常处理

捕获异常

异常捕捉可以使用 try/except 语句。
except (RuntimeError, TypeError, NameError)
被抛出的异常，应该是异常类的实例，如何获得这个对象呢？使用as子句。


一个 try 语句可能包含多个except子句，分别来处理不同的特定的异常。最多只有一个分支会被执行。
一个except子句可以同时处理多个异常，这些异常将被放在一个括号里成为一个元组。
最后一个except子句可以忽略异常的名称，它将被当作通配符使用。你可以使用这种方法打印一个错误信息，然后再次把异常抛出。
finally中一般放置资源的清理、释放工作的语句。

由于使用了try/except语句块则捕捉到了这个异常，异常生成位置之后语句将不再执行，转而执行对应的except部分的语句。

finally与return混用：
（1）不管try中是否发生异常，也不管except是否可以捕获异常，也无论try或except中是否有return，finally中的代码都必须执行。
（2）如果finally中有return，就从finally块的的return回去。此时try和catch中的return语句执行了一半，执行了第一个动作。所以，finally中的return语句会覆盖刚刚的返回值。
（3）如果finally中没有return，那么先把try或except中该执行的执行完，在return结束当前方法之前（此时return已经压栈不会被fially中的代码改变，类似i++的意思）先走一下finally，然后回去结束当前方法。

“return 返回值”语句有两个动作：
（1）把返回值放到“操作数栈”中。
（2）结束当前方法的执行，把“操作数栈”中的值会返回给调用处。

抛出异常

（1）raise显示抛出raise [Exception [, args [, traceback]]]。
　　　raise后要求应该是BaseException类的子类或实例，如果是类，将被无参实例化。
　　　raise后什么都没有，表示抛出最近一个被激活的异常，如果没有被激活的异常，则抛类型异常。这种方式很少用。
（2）解释器抛出。

raise 唯一的一个参数指定了要被抛出的异常。它必须是一个异常的实例或者是异常的类（也就是 Exception 的子类）。
如果你只想知道这是否抛出了一个异常，并不想去处理它，那么一个简单的 raise 语句就可以再次把它抛出。

异常总是向外层抛出，如果外层没有处理这个异常，就会继续向外抛出。
如果内层捕获并处理了异常，外部就不能捕获到了。
如果到了最外层还是没有被处理，就会中断异常所在的线程的执行。注意整个程序结束的状态返回值。
但是如果内层有finally且其中有return、break语句，则异常就不会继续向外抛出。
