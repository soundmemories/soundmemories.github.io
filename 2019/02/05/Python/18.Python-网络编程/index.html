Socket

Socket是网络编程的一个抽象概念。
通常用一个Socket表示“一个网络链接”，而创建一个Socket需要知道目标计算机的IP地址和端口号，再指定协议类型即可。

socket类

Python中使用socket类来创建“一个网络链接”，其本身有几个创建方法：
（1）socket.socket(family=AF_INET, type=SOCK_STREAM, proto=0, fileno=None)：创建一个socket对象。
（2）socket.socketpair(...)：创建一对socket对象。
（3）socket.fromfd(...)：创建一个socket对象，根据指定配置文件来创建。
（4）socket.fromshare(...)：创建一个socket对象，根据socket.share()的数据来创建。
创建socket主要参数：
（1）family：指定地址簇。可选AF_INET（ipv4，默认）、AF_INET6（ipv6）、AF_UNIX（Unix下使用）、AF_CAN、AF_PACKET、AF_RDS。
（2）type：socket类型。可选SOCK_STREAM（TCP，默认）、SOCK_DGRAM（UDP）、SOCK_RAW、SOCK_RDM、SOCK_SEQPACKET。
（3）proto：协议号。通常为0，可以省略。在地址簇为AF_CAN 的情况下，协议号应为CAN_RAW、CAN_BCM、CAN_ISOTP之一。
（4）fileno：如果指定了fileno，那么将从这一指定的文件描述符中自动检测family、type 和 proto 的值。
socket类还提供了一些与网络相关的方法：
（1）socket.close(fd)：关闭一个socket对象。
（2）socket.getaddrinfo(host, port, family=0, type=0, proto=0, flags=0)：将host/port参数转换为5元组的序列(family, type, proto, canonname, sockaddr)，其中包含创建socket所需的所有参数。3.7新增。
（3）socket.gethostbyname(hostname)：将主机名转换为 IPv4 地址格式。IPv4 地址以字符串格式返回。
（4）socket.gethostname()：返回一个字符串，包含当前正在运行 Python 解释器的机器的主机名。
（5）socket.getnameinfo(sockaddr, flags)：将socket地址 sockaddr 转换为二元组 (host, port)。

socket对象

socket对象可使用的方法，address是二元组（hostaddr，port）：
（1）accept()：等待传入连接。返回一个表示连接和客户端地址的新socket对象(socket object, address info)。
（2）bind(addr)：将socket对象绑定到 address。多次绑定会报错。
（3）close()：将socket对象标记为关闭，建议使用with语句。
（4）connect(addr)：连接到 address 处的远程socket对象。如果连接被信号中断，则本方法将等待，直到连接完成。
（5）getpeername()：返回socket对象连接到的远程地址。
（6）getsockname()：返回socket对象本身的地址。
（7）getsockopt(level, optname[, buflen])：返回socket对象设置信息。
（8）getblocking()：如果socket对象处于阻塞模式，返回 True，非阻塞模式返回 False。3.7新增。
（9）gettimeout()：返回socket对象操作相关的超时秒数（浮点数），未设置超时则返回 None。
（10）listen()：启动一个监听服务，用于接受连接。服务器端。
（11）recv(buflen[, flags])：从socket对象接收数据。返回值是一个字节对象，表示接收到的数据。bufsize 指定一次接收的最大数据量。
（12）recv_into(buffer[, nbytes[, flags]])：从socket对象接收至多 nbytes 个字节，将其写入缓冲区而不是创建新的字节串。
（13）recv(bufsize[, flags])：从socket对象接收数据。返回值是一对 (bytes, address)，其中 bytes 是字节对象，表示接收到的数据，address 是发送端socket对象的地址。
（14）recvfrom_into(buffer[, nbytes, [, flags])：从socket对象接收数据，将其写入 buffer 而不是创建新的字节串。返回值是一对 (nbytes, address)，其中 nbytes 是收到的字节数，address 是发送端socket对象的地址。
（15）sendall(bytes[, flags])：发送数据给socket对象。本socket对象必须已连接到远程socket对象。持续从 bytes 发送数据，直到所有数据都已发送或发生错误为止。成功后会返回 None。出错后会抛出一个异常，此时并没有办法确定成功发送了多少数据。
（16）send(bytes[, flags])：发送数据给socket对象。本socket对象必须已连接到远程socket对象。返回已发送的字节数。应用程序要负责检查所有数据是否已发送，如果仅传输了部分数据，程序需要自行尝试传输其余数据。
（17）sendfile(file, offset=0,count=None)：发送一个文件直到EOF，使用高性能的os.sendfile机制，返回发送的字节数。如果win下不支持sendfile，或者不是普通文件，使用send()发送文件。offset告诉起始位置。
（18）sendto(bytes, address)：发送给指定address信息。UDP发送数据。
（19）setblocking(flag)：设置socket对象为阻塞或非阻塞模式：如果 flag 为 false，则将socket对象设置为非阻塞，否则设置为阻塞。
（20）settimeout(value)：为阻塞socket对象的操作设置超时。如果赋为一个非零值，那么如果在操作完成前超过了超时时间 value，后续的socket对象操作将抛出 timeout 异常。如果赋为 0，则socket对象将处于非阻塞模式。如果指定为 None，则socket对象将处于阻塞模式。
（21）setsockopt(level,optname,value)：设置套接字选项的值。比如缓冲区大小。
（22）shutdown(how)：关闭一半或全部的连接。如果 how 为 SHUT_RD，则后续不再允许接收。如果 how 为 SHUT_WR，则后续不再允许发送。如果 how 为 SHUT_RDWR，则后续的发送和接收都不允许。
（23）makefile(mode='r', buffering=None, *, encoding=None, errors=None, newline=None)：创建一个与该套接字相关连的文件对象，将recv()看做读方法，将send()看做写方法。

上例中服务器端，使用了心跳机制（一定时间内无通信即可断开），客户端不需要响应。

在服务器端，可加一个心跳机制或响应机制。
心跳机制：就是一端定时，或定时发往另一端的信息，一般每次数据越少越好。心跳时间间隔约定好就行。
响应机制：一端收到另一端的消息后返回的确认信息。

一般来讲：
（1）客户端定时发往服务端的，服务端并不需要响应回复客户端，只需要记录该客户端还活着就行了。
（2）服务端定时发往客户端的，一般需要客户端响应响应来表示活着，如果没有收到响应的客户端，服务端移除其信息。这种实现较为复杂，用的较少。
（3）也可以双向都使用心跳和响应，用的更少。

SocketServer

socket编程过于底层，编程虽然有套路，但是想要写出健壮的代码还是比较困难的，所以很多语言都对socket底层API进行封装，Python的封装就是——socketserver模块。它是网络服务编程框架，便于企业级快速开发。SocketServer简化了网络服务器的编写。

创建Server对象和使用

SocketServer有4个同步类：
（1）TCPServer
（2）UDPServer
（3）UnixStreamServer
（4）UnixDatagramServer

SocketServer有2个Mixin类，用来支持异步：
（1）ForkingMixIn:
　　　class ForkingUDPServer(ForkingMixIn, UDPServer)
　　　class ForkingTCPServer(ForkingMixIn, TCPServer)
（2）ThreadingMixIn：
　　　class ThreadingUDPServer(ThreadingMixIn, UDPServer)
　　　class ThreadingTCPServer(ThreadingMixIn, TCPServer)
fork是创建多进程，thread是创建多线程。fork需要操作系统支持，Windows不支持。

以上这些类，都可以创建对应的Server对象。
一般使用BaseServer创建Server对象：
socketserver.BaseServer(server_address, RequestHandlerClass)：需要提供服务器绑定的地址信息，和用于处理请求的RequestHandlerClass类。RequestHandlerClass类必须继承BaseRequestHandler类或其子类，可重写其handle()、setup()、finish()。


Server对象的方法：
（1）serve_forever(poll_interval=0.5)：处理请求，直到shutdown()。每隔poll_interval秒检测一次是否shutdown()。
（2）shutdown()：关闭服务。
