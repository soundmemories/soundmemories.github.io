468. 验证IP地址

难度：中等
题目链接：https://leetcode.cn/problems/validate-ip-address/
题目描述：
给定一个字符串 queryIP。如果是有效的 IPv4 地址，返回 "IPv4" ；如果是有效的 IPv6 地址，返回 "IPv6" ；如果不是上述类型的 IP 地址，返回 "Neither" 。

有效的IPv4地址 是 “x1.x2.x3.x4” 形式的IP地址。 其中 0 <= xi <= 255 且 xi 不能包含 前导零。例如: “192.168.1.1” 、 “192.168.1.0” 为有效IPv4地址， “192.168.01.1” 为无效IPv4地址; “192.168.1.00” 、 “192.168@1.1” 为无效IPv4地址。

有效的IPv6地址 是 “x1:x2:x3:x4:x5:x6:x7:x8” 形式的IP地址，其中 1 <= xi.length <= 4；xi 是一个 十六进制字符串 ，可以包含数字、小写英文字母( 'a' 到 'f' )和大写英文字母( 'A' 到 'F' )；在 xi 中允许前导零。例如 "2001:0db8:85a3:0000:0000:8a2e:0370:7334" 和 "2001:db8:85a3:0:0:8A2E:0370:7334" 是有效的 IPv6 地址，而 "2001:0db8:85a3::8A2E:037j:7334" 和 "02001:0db8:85a3:0000:0000:8a2e:0370:7334" 是无效的 IPv6 地址。

打开操作

打开一个文件，返回一个文件对象(流对象)和文件描述符。打开文件失败，则返回异常。

  • file：打开或者要创建的文件名。如果不指定路径，默认是当前路径。

    • 'w'：只写打开。file不存在时新建，file存在时清空内容后打开，每次打开都是空白文件，慎用！
    • 'x'：新建file，file存在则报错，只写打开。
    • 'a'：file内容末尾追加，只写打开，file不存在则新建。
    • 'b'：二进制模式。
    • 't'：缺省的，文本模式。open的默认模式其实是'rt'。
    • '+'：给只读、只写方式追加缺失的读或者写模式。

      文本模式t：字符流，将文件的字节按照某种字符编码理解，按照字符操作。open的默认mode就是rt。

      二进制模式b：字节流，将文件就按照字节理解，与字符编码无关。二进制模式操作时，字节操作使用bytes类型。

windows中使用codepage代码页，可以认为每一个代码页就是一张编码表。cp936等同于GBK。

文件指针

文件指针，指向当前字节位置。
mode=r，指针起始在0。
mode=a，指针起始在EOF。

tell()：显示指针当前位置（字节为单位）。
seek(offset[, whence]：移动文件指针位置。offest偏移多少字节，whence从哪里开始。
- whence（文本模式）：
- 0：缺省值，表示从头开始，offest只能正整数。
- 1：从当前位置，offest只接受0，相当于原地不动，所以没什么用。
- 2：从EOF开始，只支持偏移0，相当于移动文件指针到EOF。
- whence（二进制模式）：
- 0：缺省值，表示从头开始，offest只能正整数。
- 1：从当前位置，offest可正可负。
- 2：从EOF开始，offest可正可负。

二进制模式支持任意起点的偏移，从头、从尾、从中间位置开始。

向后seek可以超界，但是向前seek的时候，不能超界，否则抛异常。

buffering

  • buffering缓冲区：
    • -1：使用缺省大小的buffer。
      • b模式时，使用io.DEFAULT_BUFFER_SIZE值，默认是4096或者8192。
      • t模式时，若是终端设备则行缓存方式，若不是则使用b模式的策略。
    • 0：
      • b模式时，关buffer。
      • t模式不支持。
    • 1：
      • b模式时，一个字节缓冲区。
      • t模式时，行缓冲，见到‘’就flush全部缓冲区数据。
    • >1：用于指定buffer大小。
      • b模式时，表示缓冲大小。设定值可以超过io.DEFAULT_BUFFER_SIZE，超出设定值才把缓冲区flush。
      • t模式时，缓冲大小是io.DEFAULT_BUFFER_SIZE个字节（缺省缓冲区大小），flush完后把当前字符串也写入磁盘。

io.DEFAULT_BUFFER_SIZE：缺省缓冲区大小（字节）。
buffer缓冲区：缓冲区一个内存空间，一般来说是一个FIFO队列，到缓冲区满了或者达到阈值，数据才会flush到磁盘。
flush()将缓冲区数据写入磁盘。
close()关闭前会调用flush()。

二进制模式，字节操作，可以指定buffer的大小。

列表解析式

  • [返回值 for 元素 in 可迭代对象 if 条件]
  • 返回一个新的列表，占用大块空间！非惰性，非迭代器。
  • 不能出现if..else，可以多个if，可以使用or、and的形式。
  • 可多层嵌套：[返回值 for 元素 in 可迭代对象 for 元素 in 可迭代对象]，相当于两层for循环！

迭代器：
> 1. 迭代器是一个可以记住遍历的位置的对象。
> 2. 迭代器对象从集合的第一个元素开始访问，直到所有的元素被访问完结束。迭代器只能往前不会后退。
> 3. 迭代器有两个基本的方法：
iter(iterable)：迭代对象包装成迭代器。迭代器是惰性的。
next(iterator[, default])：从迭代器中取元素。
> 4. 必须是迭代器才能用next方法。但可迭代对象不一定是迭代器。
> 5. 字符串，列表或元组对象都可用于创建迭代器。

生成器

生成器表达式

  • ()一次只占用一个空间！惰性！生成器对象是迭代器。
  • (返回值 for 元素 in 可迭代对象 if 条件)
  • 返回一个生成器

和列表解析式的区别：
1. 生成器表达式是按需计算（或称惰性求值、延迟计算），需要的时候才计算值。
2. 列表解析式是立即返回值。
3. 生成器表达式生成一个生成器对象，是迭代器。比如放在 for ..in后面
4. 生成器对象只能遍历一次。

range非迭代器，enumerate是迭代器，xrange是生成器（在Python3.5中range和xrange合并为range）。

优先考虑使用生成器，用到尾之前可以重置生成器循环使用。

  2. 生成器函数：
    • 函数体中包含yield语句的函数，就是生成器函数，调用后返回生成器对象；

      生成器函数等价于生成器表达式，只不过生成器函数可以写的更加复杂；

1. 在生成器函数中，使用多个yield语句，执行一
次后会暂停执行，把yield表达式的值返回；
2. 再次执行会执行到下一个yield语句；
3. return语句依然可以终止函数运行，但return语句的返回值不能被获取到；
4. return会导致当前函数返回，无法继续执行，
也无法继续获取下一个值，抛出StopIteration异常；
5. 如果函数没有显式的return语句，如果生成器
函数执行到结尾（相当于执行了return None），一样会抛出StopIteration异常；

总结下生成器函数：
1. 包含yield语句的生成器函数调用生成 生成器对象 的时候，生成器函数的函数体不会立即执行；
2. next(generator) 会从函数的当前位置向后执行到之后碰到的第一个yield语句，会弹出值，并暂停函数执行；
3. 再次调用next函数，和上一条一样的处理过程；
4. 继续调用next函数，生成器函数如果结束执行了（显式或隐式调用了return语句），会抛出StopIteration异常；

生成器的应用：作重要的作用是把函数切开执行，碰到yield暂停可以做其它事情，之后再启动就可以。

协程调度器实现思路：

  1. 有2个生成器A、 B；
  2. next(A)后，A执行到了yield语句暂停，然后去执行next(B)，B执行到yield语句也暂停，然后再次调用next(A)，再调用next(B)在，周而复始，就实现了调度的效果；
