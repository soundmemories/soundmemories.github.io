JDBC

在Java中，数据库存取技术可分为如下几类：
（1）JDBC直接访问数据库。
（2）JDO（Java Data Object）技术。
（3）第三方O/R工具，如Hibernate, Mybatis 等。
JDBC是Java访问数据库的基石，JDO, Hibernate等只是更好的封装了JDBC。

什么是JDBC？
JDBC(Java Database Connectivity)是一个独立于特定数据库管理系统（DBMS）的API、通用的SQL数据库存取和操作的公共接口（一组API），定义了用来访问数据库的标准Java类库，使用这个类库可以以一种标准的方法、方便地访问数据库资源。
JDBC是SUN公司（Oracle公司甲骨文）提供一套用于数据库操作的接口API，Java程序员只需要面向这套接口编程即可。
不同的数据库厂商，需要针对这套接口，提供不同实现。不同的实现的集合，即为不同数据库的驱动。

JDBC为访问不同的数据库提供了一种统一的途径，为开发者屏蔽了一些细节问题。
JDBC的目标是使Java程序员使用JDBC可以连接任何提供了JDBC驱动程序的数据库系统，这样就使得程序员无需对特定的数据库系统的特点有过多的了解，从而大大简化和加快了开发过程。

元编程

元编程概念来自LISP和smalltalk。
我们写程序是直接写代码，是否能够用代码来生成未来我们需要的代码吗？这就是元编程。
用来生成代码的程序称为元程序metaprogram，编写这种程序就称为元编程metaprogramming。

Python中：
（1）所有非object类都继承自object类。
（2）所有类的类型包括type类都是type。
（3）type类继承自object类，object类的类型也是type类。

Python语言能够通过反射实现元编程。

type

动态语言和静态语言最大的不同，就是函数和类的定义，不是编译时定义的，而是运行时动态创建的。
比方说我们要定义一个Hello的class，就写一个hello.py模块：


当Python解释器载入hello模块时，就会依次执行该模块的所有语句，执行结果就是动态创建出一个Hello的class对象。

type()函数可以查看一个类型或变量的类型，Hello是一个class，它的类型就是type，而h是一个实例，它的类型就是class Hello。

class的定义是运行时动态创建的，而创建class的方法就是使用type()函数。

type()函数既可以返回一个对象的类型，又可以创建出新的类型，比如，我们可以通过type()函数创建出Hello类，而无需通过class Hello(object)…的定义：


type(name, bases, dict)：返回一个name类。name为class的名称，bases为继承的父类集合（注意Python支持多重继承，如果只有一个父类，别忘了tuple的单元素写法），dict为class的方法名与函数绑定。

通过type()函数创建的类和直接写class是完全一样的，因为Python解释器遇到class定义时，仅仅是扫描一下class定义的语法，然后调用type()函数创建出class。

正常情况下，我们都用class Xxx…来定义类，但是，type()函数也允许我们动态创建出类来，也就是说，动态语言本身支持运行期动态创建类，这和静态语言有非常大的不同，要在静态语言运行期创建类，必须构造源代码字符串再调用编译器，或者借助一些工具生成字节码实现，本质上都是动态编译，会非常复杂，比如Java的反射。

除了使用type()动态创建类以外，要控制类的创建行为，还可以使用metaclass元类。

当我们定义了类以后，就可以根据这个类创建出实例，若是想用类创建类呢？那就必须用元类。
换句话说，你可以把类，看成是元类创建出来的“实例”。

模块

Python中只有一种模块对象类型。为了模块化组织模块的便利，提供了”包”的概念。

模块（module），指的是Python的源代码文件。一个Python文件源代码文件就是一个模块。
包（package），指的是模块组织在一起的和包名同名的目录及其相关文件。

如果把当前Python文件，当作模块导入，必须遵守命名规范：非数字开头的字母数字和下划线的组合，为避免冲突不要使用系统模块名。

模块的属性

（1）__name__：模块名。
（2）__dict__：模块的命名空间，也可通过dir()查看。
（3）__package__：当模块是包，同 __name__，否则，可以设置为顶级模块的空字符串。
（4）__file__：字符串，源文件路径。
（5）__cached__：字符串，编译后的字节码文件路径。
（6）__spec__：显示模块的规范。
（7）__all__：可自定义设置导入的模块或者包。如果模块没设置，from ... import *只导入非下划线开头的该模块的变量。如果设置了，则只导入__all__ 列表中指定的名称，甚至是私有模块。为避免导入过多，编写模块时，应该尽量加入 __all__。

模块搜索顺序

（1）程序主目录。程序运行的主程序脚本所在的目录。
（2）PYTHONPATH目录。环境变量PYTHONPATH设置的目录。
（3）标准库目录。Python自带的库模块所在目录。

日志记录

日志是对软件执行时所发生事件的一种追踪方式。
开发者对代码添加日志调用，借此来指示某事件的发生。
一个事件通过一些包含变量数据的描述信息来描述，每个事件发生时的数据都是不同的。
开发者还会区分事件的重要性，重要性也被称为 等级 或 严重性。

一般情况下，任务和其日志工具：
（1）对于命令行或程序的应用，结果显示在控制台：print()。
（2）在对程序的普通操作发生时提交事件报告（比如：状态监控和错误调查）：logging.info()一般情况下使用。logging.debug()当有诊断目的需要详细输出信息时使用。
（3）基于一个特殊的运行时事件，提出一个警告信息：warnings.warn()该事件是可以避免的，需要修改客户端应用以消除告警。logging.warning()不需要修改客户端应用，但是该事件还是需要引起关注。
（4）对一个特殊的运行时事件报告错误：使用异常。
（5）报告错误而不引发异常（如在长时间运行中的服务端进程的错误处理）：logging.error()或logging.exception()或logging.critical()，它们分别适用于特定的错误及应用领域。

logging

logging模块为应用与库定义了实现灵活的事件日志系统的函数与类。
logging模块最主要的好处是，所有的 Python 模块都可能参与日志输出，包括你的日志消息和第三方模块的日志消息。

logging模块采用模块化方法，并提供几类组件：
（1）记录器：暴露了应用程序代码直接使用的接口。
（2）处理程序：将日志记录（由记录器创建）发送到适当的目标。
（3）过滤器：提供了更精细的附加功能，用于确定要输出的日志记录。
（4）格式化程序：指定最终输出中日志记录的样式。
